# V1 Hard Filter 정책 문서

> **문서 목적**: 종목 추천 유니버스에서 사전 제외해야 할 종목의 기준을 정의하고, V1 운영 가능한 권장 수치 기준을 제안한다.
> 이 문서는 권장안이며, 각 항목의 최종 수치 확정은 운영자의 판단이 필요하다.
> 코드 구현, 데이터 수집, 점수 계산은 이 문서에서 다루지 않는다.

---

## 1. Hard Filter 설계 원칙

Hard Filter는 **점수로 극복할 수 없는 제외 기준**이다. 아무리 다른 평가축에서 높은 점수를 받아도 Hard Filter를 통과하지 못하면 메인 추천과 same_sector_alternatives 모두에서 제외된다.

Hard Filter의 목적은 세 가지다:
1. **독자 보호**: 재무적·구조적 위험이 명확한 종목을 필터링한다.
2. **서비스 신뢰 보호**: 관리종목, 이슈 종목이 추천되면 서비스 신뢰가 훼손된다.
3. **운영 단순화**: 사람의 매번 판단이 없어도 자동으로 걸러야 하는 기준을 명확히 한다.

Hard Filter는 두 등급으로 나뉜다:

| 등급 | 처리 방식 | 설명 |
|------|----------|------|
| **즉시 제외 (Hard Block)** | 확인 즉시 후보에서 삭제 | 예외 없음. 점수 계산 없이 제거. |
| **추가 검토 필요 (Soft Flag)** | 플래그 후 admin 판단 | 자동 제외는 아니나 admin이 반드시 검토해야 함. |

## 2. Hard Filter 항목 목록

### 2-1. 공식 지정 이슈 종목 (즉시 제외)

| 항목 | 정의 | 적용 등급 | 개별주/ETF |
|------|------|---------|-----------|
| 관리종목 | 한국거래소 지정 관리종목 | Hard Block | 동일 적용 |
| 거래정지 종목 | 현재 거래정지 상태 | Hard Block | 동일 적용 |
| 감사의견 비적정 | 감사보고서 의견이 한정/부적정/의견거절인 경우 | Hard Block | 동일 적용 |
| 투자경고 종목 | 한국거래소 지정 투자경고 종목 | Hard Block | 동일 적용 |
| 투자위험 종목 | 한국거래소 지정 투자위험 종목 | Hard Block | 동일 적용 |
| 단기과열 종목 | 한국거래소 지정 단기과열 종목 (지정 기간 동안) | Hard Block | 동일 적용 |
| 투자주의 종목 | 한국거래소 지정 투자주의 종목 | Soft Flag | 동일 적용 |

**이유**: 거래소가 공식으로 지정한 이슈 종목은 규칙 기반 확인이 가능하고, 해당 지정 자체가 독자에게 설명하기 어려운 위험 신호다. 투자주의는 경고보다 낮은 수준이므로 Soft Flag으로 처리해 admin 판단을 거친다.

### 2-2. 영업 적자 기업

**항목 정의**: 최근 일정 기간 영업이익 기준으로 지속적인 적자 상태인 기업.

**권장안**: 최근 4분기(TTM, Trailing Twelve Months) 합산 영업이익이 음수(-)인 기업은 **Soft Flag** 처리.

| 기준 | 처리 등급 | 설명 |
|------|---------|------|
| TTM 영업이익 합산 음수 | Soft Flag | admin 검토 후 판단 |
| 최근 2개 분기 연속 영업이익 음수 + TTM 합산도 음수 | Hard Block | 지속 적자 기업으로 즉시 제외 |

**권장안 이유**:
- 단일 분기 적자만으로 Hard Block 하면, 계절성 업종(예: 여름 한정 매출 구조)이나 일시적 대규모 투자 비용으로 1개 분기 적자가 난 기업도 걸러진다. 1개월형 서비스는 턴어라운드 기업을 잡는 관점이 있으므로 TTM 기준이 더 적합하다.
- 단, 최근 2개 분기 연속 적자 + TTM 합산도 음수라면 구조적 문제 가능성이 높으므로 Hard Block으로 강화한다.

**ETF 적용 여부**: ETF는 영업이익 개념이 없으므로 이 필터는 ETF에 적용하지 않는다. 다만 ETF 운용사의 재정 건전성이 문제가 되는 극단적 케이스는 운용사 신뢰도 문제로 별도 처리한다.

**바이오/헬스케어 예외 검토**:
- 바이오 기업은 구조적으로 R&D 단계에서 영업적자가 발생한다.
- 권장안: 바이오/헬스케어(`HEALTHCARE` 섹터) 종목에 한해 TTM 영업이익 적자 항목을 Soft Flag으로 완화 적용. 단, 현금소진율(Burn Rate)이 심각한 수준이면 Hard Block 유지.

> ⚠️ **최종 확정 필요**: "최근 2개 분기"의 정확한 기준 기간, 바이오 예외 범위 수치 기준은 운영자 확인 필요.

### 2-3. 신규 상장 종목

**항목 정의**: 주식 시장 상장 후 일정 기간이 지나지 않은 종목.

**권장안**: 상장일로부터 **6개월 미만** 종목은 Hard Block.

**권장안 이유**:
- 상장 직후 6개월은 보호예수(Lock-up) 기간이 순차 만료되는 구간이다. 대주주 및 기관 보호예수 물량의 매도 압력이 주가에 직접 영향을 미치므로 1개월형 추천에서 가격 예측 신뢰도가 낮다.
- 6개월이 지나면 최소 1~2회 분기 실적 발표가 이루어지므로, 공개된 재무 정보가 존재한다.
- 상장 직후 과도한 공모가 대비 급락 리스크를 피할 수 있다.

**ETF 적용 여부**: ETF는 상장일 기준 6개월 제외를 동일하게 적용한다. 다만 기존에 장외에서 운용된 운용 이력이 있는 ETF(예: 공모펀드 → ETF 전환)는 예외 검토 가능.

> ⚠️ **최종 확정 필요**: 6개월은 권장이며 3개월 vs 6개월 vs 12개월 중 운영자 판단 필요. 스팩(SPAC) 합병 종목의 경우 상장일 기산 방식도 별도 검토 필요.

### 2-4. 동전주 (저가 종목)

**항목 정의**: 주가 절대 금액이 일정 기준 이하인 종목.

**권장안**: 주가(종가 기준) **2,000원 미만** 종목은 Hard Block.

**권장안 이유**:
- 한국 시장에서 2,000원 미만 종목은 1호가 단위(1원)가 수익률에 미치는 영향이 크고, 호가 스프레드 비율이 상대적으로 높아 일반 투자자 거래 비용이 증가한다.
- 2,000원 미만 구간은 단기 테마성 급등락이 빈번하고, 이 서비스의 1개월형 관점에서 신뢰할 수 있는 가격 구간 설정이 어렵다.
- 1,000원 미만은 사실상 "투기적 동전주"로 간주되며, 2,000원은 그보다 약간 넓은 안전 마진을 준다.

**ETF 적용 여부**: ETF는 동전주 필터를 동일하게 적용하지 않는다. 채권형/배당형 ETF는 NAV 특성상 1,000~2,000원대 가격 구조를 가질 수 있기 때문이다. 단, 1,000원 미만 ETF는 Soft Flag으로 처리한다.

> ⚠️ **최종 확정 필요**: 1,000원 vs 2,000원 vs 3,000원 중 운영자 확인 필요. 시장 상황에 따라 기준 주가가 상대적으로 달라질 수 있음.

### 2-5. 저유동성 종목

**항목 정의**: 일정 기간 일평균 거래대금이 기준 이하인 종목.

> **[보정 이유]**: 이전 권장안에서 개별주 단일 기준(30억)을 사용했으나, KOSPI와 KOSDAQ는 시장 구조상 평균 거래대금 수준이 다르다. 단일 기준은 KOSPI에서는 지나치게 관대하고 KOSDAQ에서는 지나치게 엄격하다. 따라서 상장 시장별로 기준을 분리한다.

**보정된 권장안**:

> 기준: 최근 20거래일 일평균 거래대금.

| 시장 | Hard Block 기준 | Soft Flag 구간 |
|------|---------------|--------------|
| KOSPI 개별주 | 일평균 거래대금 **50억 원 미만** | 50억~100억 원 |
| KOSDAQ 개별주 | 일평균 거래대금 **20억 원 미만** | 20억~30억 원 |
| ETF | 일평균 거래대금 **10억 원 미만** | 10억~20억 원 |

**시장별 분리 이유**:
- **KOSPI**: 코스피 종목은 시총 기준으로 이미 일정 규모 이상이다. 코스피 상장 종목 중 일평균 거래대금이 50억 원 미만인 경우는 유동성 면에서 실질적 거래 불가에 가깝다. 30억 기준은 코스피 종목에 적용하기에 너무 낮다.
- **KOSDAQ**: 코스닥은 중소형·성장주 위주로 평균 거래대금이 코스피보다 낮다. 30억을 그대로 적용하면 정상적으로 거래 가능한 코스닥 중형주도 다수 걸러진다. 20억으로 하향 조정이 적합하다.
- **ETF**: LP(유동성공급자)가 호가를 공급하므로 거래대금이 낮아도 실제 매매는 가능하다. 10억 기준을 유지한다.

**개별주 공통 원칙**:
- 서비스 독자는 개인 투자자 위주이다. 수십~수백만 원 포지션 기준으로 시장가 진입·청산이 가능한 최소 유동성을 확보하는 것이 목적이다.
- 추천 종목은 독자가 실제로 "거래 가능한 아이디어"여야 한다는 서비스 원칙을 유동성 기준이 보호한다.

> ⚠️ **최종 확정 필요**: KOSPI 50억, KOSDAQ 20억 기준은 서비스 타겟 독자층의 평균 투자 규모와 거래 방식에 따라 달라진다. 운영 초기 결과를 보고 조정한다. ETF LP 활동 수준에 따른 10억 기준도 주기적으로 검토 필요.

### 2-6. 최근 급등 과열 구간 종목

> **[보정 이유]**: 이전 권장안에서 5거래일 +20% 초과와 20거래일 +50% 초과를 Hard Block으로 설정했으나, 1개월형 서비스 특성을 고려하면 이 기준은 과도하게 엄격하다. 급등 자체가 문제가 아니라 "촉매 없는 급등"이 문제다. 따라서 보정된 권장안은 극단적 급등만 Hard Block으로 유지하고, 나머지는 Soft Flag + admin 검토로 전환한다.

**항목 정의**: 단기간에 비정상적으로 급등하여 추가 상승 여력보다 되돌림 위험이 큰 상태의 종목.

**보정된 권장안**:

| 기준 | 처리 등급 | 설명 |
|------|---------|------|
| 최근 5거래일 등락률 +40% 초과 | Hard Block | 극단적 단기 급등. 어떤 촉매로도 정상 추천 불가. |
| 최근 20거래일 등락률 +80% 초과 | Hard Block | 한 달 내 2배 수준. 추가 선반영 위험 매우 높음. |
| 최근 5거래일 등락률 +20~40% | Soft Flag | admin이 촉매 유효성 및 가격 위치 확인 필요. |
| 최근 20거래일 등락률 +30~80% | Soft Flag | admin이 촉매 지속성 및 추가 여력 확인 필요. |

**1개월형 서비스에서 Hard Block 기준을 좁힌 이유**:

- **1개월형 추천에서 모멘텀은 유효한 신호다**: 5거래일 +20%는 실적 서프라이즈, 정책 발표, 대형 계약 공시 등 실체 있는 촉매에서 충분히 발생한다. 이 경우 이후 2~3주에도 추가 반응이 이어질 수 있다. Hard Block으로 차단하면 정작 가장 강한 모멘텀 종목을 놓친다.
- **1개월형의 진짜 리스크는 "촉매 없는 급등"이다**: 핵심 질문은 "얼마나 올랐는가"가 아니라 "왜 올랐는가, 그 이유가 아직 유효한가"다. 이 판단은 숫자 하나로 결정할 수 없으므로 Soft Flag + admin 검토가 더 적합하다.
- **거래소 단기과열 지정이 이미 Hard Block으로 존재한다**: 거래소가 공식으로 단기과열을 지정한 경우는 이미 2-1항에서 Hard Block으로 처리된다. 급등 수치 기준의 Hard Block은 이를 보완하는 역할이므로, 극단적 수준(+40%, +80%)에만 적용하면 충분하다.

**Soft Flag 처리 시 admin 검토 의무 항목**:
- 급등의 구체적 원인(공시, 뉴스, 테마)이 확인되는가?
- 급등 원인이 추천 사유와 연결되는가, 아니면 이미 소화된 재료인가?
- 현재 가격 위치가 관심 구간(watch zone) 내에 있는가?
- 급등 이후 되돌림 없이 고점 유지 중이라면 추가 상승 여력이 있는가?

**ETF 적용 여부**: 일반 ETF에도 동일하게 적용한다. 단, 레버리지 ETF는 기초 지수 등락률로 환산하여 판단한다. (예: 2배 레버리지 ETF가 20거래일 +60% 상승 시, 기초 지수 기준으로는 +30%이므로 Soft Flag 구간으로 처리)

> ⚠️ **최종 확정 필요**: Hard Block 임계값(5일 +40%, 20일 +80%)은 시장 변동성 수준에 따라 조정 가능하다. Soft Flag 구간의 하한(5일 +20%, 20일 +30%)도 운영 초기 결과를 보고 조정한다. 단, 어느 경우든 Hard Block 기준을 Soft Flag 기준보다 낮게 설정하는 역전은 금지.

## 3. Hard Filter 항목 요약표

| 항목 | 권장 기준 | 처리 등급 | 개별주 | ETF |
|------|----------|---------|--------|-----|
| 관리종목 | 거래소 지정 여부 | Hard Block | ✅ | ✅ |
| 거래정지 | 현재 상태 | Hard Block | ✅ | ✅ |
| 감사의견 비적정 | 한정/부적정/의견거절 | Hard Block | ✅ | ✅ |
| 투자경고 | 거래소 지정 여부 | Hard Block | ✅ | ✅ |
| 투자위험 | 거래소 지정 여부 | Hard Block | ✅ | ✅ |
| 단기과열 지정 | 거래소 지정 기간 중 | Hard Block | ✅ | ✅ |
| 투자주의 | 거래소 지정 여부 | Soft Flag | ✅ | ✅ |

## 4. Soft Flag 운영 방식

Soft Flag가 붙은 종목은 자동 제외되지 않는다. 아래 절차로 처리한다.

1. Draft 생성 시 Soft Flag 종목 목록이 별도 표시된다.
2. Admin 검수 화면에서 해당 종목의 플래그 사유가 명시된다.
3. Admin이 플래그 사유를 확인하고, 해당 종목을 포함할 경우 포함 사유를 검수 메모에 기록한다.
4. 기록 없이 Soft Flag 종목을 그냥 통과시키는 것은 원칙적으로 금지한다.
