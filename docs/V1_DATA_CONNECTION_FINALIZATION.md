# V1 데이터 연결 방식 최종 확정

> **문서 목적**: V1 기준으로 KRX / DART / ECOS / FRED / 뉴스 신호의 역할을 최종 확정하고,
> 각 데이터 소스의 책임 범위·필드 매핑·수집 우선순위를 설계 차원에서 최종 결론으로 기록한다.
>
> 이 문서는 Phase B-2 구현 착수 직전 마지막 설계 확정 문서다.
> 코드 구현·API 연동·스크립트 작성은 이 문서에서 하지 않는다.
>
> 이전 설계 문서들(V1_DATA_SOURCE_MAP, V1_DATA_SOURCE_STRATEGY, V1_DATA_SOURCE_SHORTLIST,
> V1_KRX_DART_ACCESS_POLICY, V1_NEWS_SIGNAL_DATA_POLICY)을 통합하여
> 미확정으로 남아 있던 항목을 이 문서에서 최종 결론으로 전환한다.

## 1. 데이터 소스별 최종 역할 정의

### 1-1. 소스별 책임 최종 확정 표

| 소스 | 최종 담당 데이터 영역 | V1 자동 수집 가능 여부 | 수동 보완 필요 영역 |
|------|--------------------|---------------------|-----------------|
| **KRX** | 시세·거래대금·지수·업종지수·수급·거래소지정현황·ETF기본메타·상장정보 | ✅ 자동 수집 가능 | 섹터 매핑 테이블(초기 1회 수동 설정), 발행 직전 지정 현황 재확인 |
| **DART** | 재무제표(영업이익·부채비율·매출)·공시목록·감사의견 | ✅ 가능(분기 갱신) | 중요 공시 판단, 어닝 서프라이즈 여부 판단 |
| **ECOS** | USD/KRW 환율·한국은행 기준금리 | ✅ 자동 수집 가능 | BOK 스탠스(매파/비둘기파) 방향 판단 |
| **FRED** | 미국 10년 국채금리·미연준 기준금리(FFR) | ✅ 자동 수집 가능 | FOMC 스탠스 방향 판단 |
| **글로벌 지수 보조** | S&P 500·NASDAQ·VIX·Nikkei 225 주간 등락률 | ✅ 가능 (보조 소스) | 수치 교차 확인 권장 |
| **뉴스 신호** | 섹터·시장·종목·ETF 방향 신호 (보완 역할) | ⚠️ V1 Phase B에서는 수동 입력. Phase C에서 자동화 설계 | 발행 전 admin 검수 필수 |

> **핵심 경계 원칙**: 위 소스 간 책임이 겹치는 항목은 없어야 한다.
> 같은 수치를 두 소스에서 이중으로 수집하지 않는다. 소스를 바꿔야 할 경우 이 문서를 갱신한다.

- 환율·금리 → ECOS·FRED 담당
- 글로벌 지수·VIX → 보조 소스 담당

## 3. DART — 최종 책임 범위

### 3-1. DART가 단독 담당하는 데이터 항목 (최종 확정)

| 데이터 항목 | 사용 목적 | 갱신 주기 |
|-----------|---------|---------|
| TTM 영업이익 (최근 4분기 합산) | Hard Filter 2-2절 적자 필터 | 분기 1회 |
| 최근 분기 영업이익·직전 분기 영업이익 | Hard Filter 2-2절 연속 적자 판단 | 분기 1회 |
| 부채비율 (D/E Ratio) | 재무 안전성 평가 | 분기 1회 |
| 매출액 전년 동기 대비 성장률 | 성장성 방향 판단 보완 | 분기 1회 |
| 감사의견 코드 | Hard Filter 2-1절 감사의견 Hard Block | 연간 (감사보고서 기준) |
| 최근 공시 목록 (종목별) | 단기 촉매 평가, 공시 이슈 확인 | 주 1회 |
| 실적 발표 예정일 | 단기 촉매 평가 | 분기 |

**DART 재무제표 기준 최종 확정 (이전 미확정 항목 해소)**:
- **별도 재무제표 기준**으로 확정한다.
- 이유: 유니버스는 KOSPI/KOSDAQ 상장 개별 법인이 대상이다. 연결 재무제표는 자회사 손익이 혼입되어 개별 법인 적자 판단이 왜곡될 수 있다.
- 예외: 지주사 구조이고 별도 재무제표가 의미 없는 종목(예: 순수 지주사)은 운영자가 개별 판단 후 유니버스에서 조정한다.

**ETF 구성 종목 데이터 소스 최종 확정 (이전 미확정 항목 해소)**:
- **KRX ETF 기본 정보 우선 → KOFIA 펀드 공시 보완** 구조로 확정한다.
- KRX에서 AUM·TER·기초지수명 수집. 구성 종목 상위 보유 비중은 KOFIA 펀드 공시에서 보완.
- 두 소스에서 동일 항목이 제공되는 경우 KRX 우선.

### 3-2. DART 접근 방식 최종 확정

- DART OpenAPI 직접 사용 (API 키 발급, 무료)
- 재무 데이터: 종목당 분기 4건 × 유니버스 규모. 일 호출 한도(10,000건) 이내에서 충분히 처리 가능.
- TTM 계산: 직전 4개 분기 개별 영업이익을 합산하는 별도 계산 로직이 구현 단계에서 필요하다.
- 공시 목록: 주 1회 종목별 최근 N건 조회.
- 호출 전략: 유니버스 전체를 한 번에 호출하지 않고 종목당 간격을 두어 순차 호출.

## 7. 뉴스 신호 — V1 자동화 수준 최종 확정

### 7-1. V1 Phase B에서 뉴스 신호의 위치

뉴스 신호는 **공식 수치 데이터의 보완 신호**이다. 다음 원칙은 변하지 않는다:
- 뉴스 신호만으로 섹터 방향성(favored/cautious)을 결정하지 않는다.
- 뉴스 신호는 발행 차단 조건이 아니다.
- 뉴스가 부족해도 수치 기반 발행이 가능해야 한다.

### 7-2. V1 Phase B에서 뉴스 신호 처리 방식 (최종 확정)

| 처리 방식 | V1 Phase B 결정 |
|---------|--------------|
| **수집 방법** | 운영자가 주간 draft 작성 전 주요 뉴스 모니터링 후 구조화 입력 (수동) |
| **저장 형식** | V1_NEWS_SIGNAL_DATA_POLICY.md에 정의된 파일 구조 그대로 사용 |
| **자동화 수준** | Phase B에서는 수동 입력. 자동/반자동 생성(크롤러·LLM 연동)은 Phase C로 유보. |
| **admin 개입** | is_approved / is_discarded 결정 + 필요 시 validation_note 기록 |
| **발행 차단 여부** | 뉴스 신호가 없어도 발행 차단하지 않는다. signal_review.json은 참고 입력. |

**자동화를 Phase C로 유보하는 이유**:
- Phase B-2의 핵심은 수치 데이터(KRX·DART·ECOS·FRED) 연결이다.
- 뉴스 자동화는 별도 도구(크롤러·LLM)가 필요하며, 수치 파이프라인이 먼저 안정화된 후 추가하는 것이 올바른 순서다.
- V1에서 수동 입력으로도 서비스 운영이 가능해야 한다는 원칙에 부합한다.

## 13. Self-check before Phase B-2

### 이번 문서에서 최종 확정한 데이터 연결 범위
- KRX: 시세·수급·지수·거래소지정·ETF기본·상장 → 파일다운로드(Day 1) + 공식API 병행 신청
- DART: 재무(별도 기준)·공시·감사의견 → OpenAPI 직접 사용
- ECOS: USD/KRW + BOK 기준금리 2개 지표
- FRED: US 10Y + FFR 2개 지표
- 보조 소스: Yahoo Finance (글로벌 지수·VIX, Hard Filter 제외)
- 뉴스 신호: Phase B에서 수동 입력 유지, Phase C에서 자동화 설계

### 아직 최종 구현으로 넘기지 않은 범위
- KRX 파일 형식 및 파싱 규칙 (직접 확인 필요)
- DART Rate Limit 실제 수치 (공식 문서 확인 필요)
- 실적 발표 예정일 구조화 수집 방식 (API 테스트 필요)
- 뉴스 자동화 도구 및 방법 (Phase C)
- 어닝 서프라이즈 컨센서스 데이터 소스 (유료 여부 미확인)
- KRX 업종지수 → 서비스 13섹터 매핑 테이블 실제 내용

### 소스별 책임이 겹치지 않도록 어떻게 정리했는지
- 시세·수급·지정현황: KRX 단독 담당으로 확정. DART와 중복 없음.
- 재무·공시·감사의견: DART 단독 담당으로 확정. KRX와 중복 없음.
- 환율: ECOS 단독 (KRX·DART는 환율 미제공 영역)
- 금리(미국): FRED 단독 (ECOS는 국내 금리 담당)
- 글로벌 지수: 보조 소스 담당. 이 영역은 KRX·DART·ECOS·FRED 모두 미제공.

### 뉴스 신호가 보완 신호 원칙을 유지하는지
- 수치 자동 수집(P1)이 뉴스 신호보다 구현 선행으로 확정됨.
- 뉴스가 없어도 수치 기반 발행 구조가 먼저 성립한다.
- 뉴스 신호는 P3(Phase C 유보)로 분류하여 보완 신호 위치를 명확히 했음.

### 운영자 개입을 최소화하도록 어떻게 정리했는지
- P1 전체(시세·수급·재무·지수·환율·금리)를 자동 수집으로 확정.
- 운영자 수동 입력은 판단이 개입되는 항목(스탠스 방향·섹터 판단·시장 요약 문구·관심 구간)만으로 한정.
