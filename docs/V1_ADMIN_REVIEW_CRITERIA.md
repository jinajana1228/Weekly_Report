# V1 Admin 검수 기준 문서

> **문서 목적**: Admin이 draft 에디션을 검수할 때 무엇을 어떤 기준으로 확인해야 하는지, 언제 승인/보류/반려를 결정해야 하는지 정의한다.
> 이 문서는 자동화 구현이나 코드 작성 없이 사람이 수행하는 검수 판단 기준을 다룬다.

---

## 1. Admin 검수의 목적

Admin 검수는 단순히 오탈자를 확인하는 작업이 아니다. 아래 3가지를 보장하는 게이트다.

1. **로직 일관성 보장**: 시장 분석과 종목 추천이 같은 방향을 가리키는가. 시장이 특정 섹터를 경계한다고 쓰면서 그 섹터 종목을 메인에 넣는 모순이 없는가.
2. **운영 정책 준수 확인**: 섹터 중복 금지, 중복 추천 정책, 제외 필터 기준이 지켜졌는가.
3. **독자 보호**: 과도한 투자 확신 표현, 리스크 누락, 설명 불충분이 없는가.

---

## 2. 검수 진행 순서

검수는 아래 순서로 진행한다. 항목 순서 자체가 중요도 순이기도 하다.

```
1단계: 운영 정책 준수 확인 (정책 위반 시 즉시 반려)
2단계: 로직 일관성 확인 (시장 분석 ↔ 추천 일관성)
3단계: 개별 종목 검수 (추천 사유, 리스크, 데이터 일치)
4단계: ETF 추가 항목 확인 (ETF 포함 시)
5단계: 문구 표현 검수 (과장 표현, 확신 표현 여부)
6단계: 최종 발행 여부 결정 (승인 / 보류 / 반려)
```

---

## 3. 1단계: 운영 정책 준수 확인 체크리스트

이 단계에서 하나라도 위반이 확인되면, 나머지 검수 이전에 즉시 반려 또는 수정 요청한다.

### 3-1. 섹터 중복 여부

| 확인 항목 | 확인 방법 |
|----------|----------|
| 메인 5개 종목의 섹터 코드가 모두 다른가? | picks[].sector 값 5개 대조 |
| 섹터 코드가 V1_SECTOR_TAXONOMY.md의 13개 코드 중 하나인가? | 허용 코드 목록 대조 |
| ETF와 동일 섹터의 개별주가 동시에 메인에 있지는 않은가? | ETF 섹터와 개별주 섹터 교차 확인 |

### 3-2. 중복 추천 정책 준수 여부

| 확인 항목 | 확인 방법 |
|----------|----------|
| 메인 5개 중 직전 주(n-1) 메인 추천과 겹치는 종목이 있는가? | admin/overlap_history.json 직전 에디션과 대조 |
| 메인 5개 중 n-2와 n-1 모두에서 추천된 종목이 있는가? (3주 연속 금지) | overlap_history.json 2개 에디션과 동시 대조 |
| 위반이 있다면 예외 승인 기록이 있는가? | draft 메모 또는 admin 검수 메모 확인 |

### 3-3. 제외 필터 준수 여부

| 확인 항목 | 확인 방법 |
|----------|----------|
| 현재 관리종목 지정 종목이 포함되지 않았는가? | 거래소 공시 확인 |
| 현재 거래정지 종목이 포함되지 않았는가? | 거래소 공시 확인 |
| 감사의견 비적정 종목이 포함되지 않았는가? | 최근 감사보고서 확인 |

> ⚠️ **미확정**: 영업적자 기준, 신규상장 기간, 동전주/저유동성 수치 기준은 정책 확정 전까지 admin이 정성적으로 판단한다.

---

## 4. 2단계: 로직 일관성 확인 체크리스트

### 4-1. 시장 분석 ↔ 추천 종목 간 논리 일관성

| 확인 항목 | 통과 기준 |
|----------|----------|
| 시장 분석에서 `cautious_sectors`로 분류된 섹터의 종목이 메인에 있다면, 해당 추천 사유가 시장 분위기를 극복하는 구체적 근거를 담고 있는가? | "이 종목만이 가진 이유"가 명시되어야 함 |
| 시장 분석의 전반적 기조(Risk-on/off)와 추천 종목의 성격(성장주/방어주)이 충돌하지 않는가? | 충돌 시 추천 사유에 설명 필요 |
| 시장 분석 헤드라인과 추천 섹터 구성이 독자에게 모순되게 읽히는 배치가 아닌가? | 전체 리포트를 독자 관점에서 한 번에 읽어보기 |

---

## 5. 3단계: 개별 종목 검수 체크리스트

메인 5개 종목 각각에 대해 아래 항목을 확인한다.

### 5-1. 추천 사유와 데이터 일치 여부

| 확인 항목 | 통과 기준 |
|----------|----------|
| `one_line_reason`에 기술된 추천 이유가 실제 확인 가능한 사실에 근거하는가? | 확인 불가능한 추측이나 루머 기반이면 통과 불가 |
| `stance`(투자 관점)에 기술된 내용이 현재 시장 상황과 모순되지 않는가? | 리포트 작성 시점 기준으로 판단 |
| `price_zone`(가격 참고 구간)이 실제 현재 주가와 현저히 다르지 않는가? | 기준가가 현실과 크게 벗어나면 수정 요청 |
| `catalysts_2_to_4_weeks`의 촉매 항목이 2~4주 내 실현 가능한 이벤트인가? | "언젠가 좋아질 것" 수준의 막연한 촉매는 통과 불가 |

### 5-2. 리스크 설명 누락 여부

| 확인 항목 | 통과 기준 |
|----------|----------|
| `risks` 항목이 비어있지 않은가? | 모든 추천 종목에 리스크가 최소 1개 이상 기재되어야 함 |
| `risks`에 기술된 리스크가 해당 종목에 실제로 해당하는 내용인가? | 형식적으로 채운 것이 아니라 이 종목의 실제 약점이어야 함 |
| 최근 이슈(규제, 실적 하락, 수급 이탈 등)가 리스크에 반영되었는가? | 공개된 부정적 신호를 의도적으로 누락해서는 안 됨 |
| 리스크 기술이 "리스크가 있을 수 있습니다" 수준의 무의미한 표현이 아닌가? | 구체적 위험 요인이 명시되어야 함 |

---

## 6. 4단계: ETF 추가 항목 확인 체크리스트

ETF가 메인 추천에 포함된 경우, 개별주 검수 항목에 더해 아래를 추가로 확인한다.

| 확인 항목 | 통과 기준 |
|----------|----------|
| 추종 지수/전략이 명확히 기술되었는가? | "미국 대형주 지수 추종" 수준이 아닌 구체적 지수명 포함 |
| 환헤지 여부가 명시되어 있는가? | 해외 ETF는 반드시 환헤지 여부 기술 |
| 레버리지/인버스 여부가 명시되어 있는가? | 레버리지 ETF라면 일반 종목과 다른 위험 구조 설명 필요 |
| 주요 구성 종목 또는 비중이 기술되었는가? | 독자가 실제 노출을 이해할 수 있는 수준의 정보 |
| 운용사 및 수수료(보수)가 기술되었는가? | 유사 ETF 간 비교 가능성 제공 |
| ETF 전용 리스크(괴리율, 유동성, 환율 등)가 별도 기술되었는가? | 개별주와 다른 ETF 고유 리스크가 있어야 함 |
| 이번 주 이 ETF를 추천하는 이유가 구체적으로 기술되었는가? | "분산 효과"만으로는 불충분. 이번 2~4주에 왜 이 ETF인지가 있어야 함 |

---

## 7. 5단계: 문구 표현 검수 체크리스트

### 7-1. 과도한 확신 표현 방지 원칙

아래 유형의 표현이 포함된 문장이 있으면 수정 요청한다.

| 금지 표현 유형 | 예시 | 대체 방향 |
|-------------|------|----------|
| 투자 결과를 확신하는 표현 | "반드시 오른다", "틀림없이 반등한다" | "반등 가능성이 있는 구간", "상승 시도 가능성" |
| 특정 수익률을 암시하는 표현 | "2~3주 내 10% 이상 수익 예상" | "긍정적 가격 반응 기대", 수익률 숫자 기재 금지 |
| 즉각적 매수를 촉구하는 표현 | "지금 당장 매수해야 한다", "지금이 기회다" | "관심 구간 진입 시 분할 접근 고려" |
| 시장 전체 방향을 단정하는 표현 | "이번 주 증시는 상승한다" | "상승 시도 가능성이 있는 구간" |
| 특정 기간 내 결과를 보장하는 표현 | "4주 내에 결과가 나온다" | "2~4주 내 가격 반응 여부를 확인하는 관점" |

### 7-2. 표현이 모호한 경우

| 확인 항목 | 통과 기준 |
|----------|----------|
| "좋은 기업이다", "성장 잠재력이 있다" 수준의 막연한 추천 사유가 있는가? | 구체적 근거가 있어야 함 |
| 추천 사유가 이 서비스의 1개월 관점과 무관한 장기 논리로만 구성되지 않았는가? | 2~4주 내 근거가 있어야 함 |

---

## 8. 중복 추천 예외 승인 시 확인 항목

직전 주 메인 추천 종목을 이번 주에도 다시 메인에 포함시키는 예외를 승인할 경우, 아래 항목을 반드시 확인하고 기록한다.

| 확인 항목 | 기준 |
|----------|------|
| 예외 허용의 구체적 사유가 있는가? | "지난주보다 더 강한 촉매가 생겼다" 수준의 구체적 이유 필요 |
| 해당 종목이 직전 주 이후 실질적인 변화가 있었는가? | 단순 지속 보유 의견이 아닌, 새로운 정보나 이벤트가 있어야 함 |
| 3주 연속 추천 해당 여부를 확인했는가? | n-2 에디션에서도 추천 여부 확인 필수 (3주 연속 시 예외 불가) |
| 예외 승인 사유를 검수 메모에 기록했는가? | 기록 없는 예외 승인은 불가 |

---

## 9. 발행 보류(on_hold)가 필요한 상황

아래 상황에서는 즉시 승인/반려를 결정하지 말고 "보류"로 처리한다.

| 상황 | 보류 이유 |
|------|---------|
| 추천 종목 중 하나에 예상치 못한 공시(실적 하향, 불공정거래 조사 등)가 발생했으나 정보가 불완전한 경우 | 추가 확인 후 판단 |
| 시장 분석의 핵심 변수(예: FOMC 결과)가 리포트 발행 직전 발표될 예정인 경우 | 발표 결과 확인 후 시장 분석 업데이트 필요 여부 검토 |
| 기재된 가격 참고 구간과 현재 실제 가격의 괴리가 크지만, 그 이유가 명확하지 않은 경우 | 시장 급변인지 데이터 오류인지 확인 필요 |
| 종목 추천 사유가 있으나, 관련 뉴스 또는 공시 원문 확인이 아직 안 된 경우 | 원문 확인 후 재검토 |

---

## 10. 반려(rejected)가 필요한 상황

아래 상황은 수정 요청이 아닌 반려(전면 재작성 요청)로 처리한다.

| 상황 | 반려 이유 |
|------|---------|
| 섹터 중복이 있고 수정으로 해결되지 않는 경우 (전면 재구성 필요) | 정책 위반 |
| 3주 연속 추천 위반이 발견된 경우 | 정책 위반, 예외 없음 |
| 리스크 설명이 메인 5개 종목 중 2개 이상에서 완전히 누락된 경우 | 독자 보호 실패 |
| 추천 사유가 확인 불가능한 루머 또는 미공개 정보처럼 보이는 경우 | 법적 리스크 |
| 가격 참고 구간의 오류가 명백하고 전반적인 데이터 신뢰가 어려운 경우 | 전면 재확인 필요 |
| ETF 추천인데 환헤지, 레버리지 여부 등 핵심 항목이 완전히 빠진 경우 | 독자 보호 실패 |

---

## 11. 운영상 Fallback 판단 포인트

검수 과정에서 기준이 불분명한 상황이 발생할 경우, 아래 원칙으로 판단한다.

| 상황 | Fallback 원칙 |
|------|-------------|
| 필터 기준이 미확정 상태에서 판단이 모호한 종목 | "독자가 이 사실을 알면 추천 종목으로 수용할 것인가?"를 기준으로 판단 |
| 시장 분석과 추천이 미묘하게 충돌하는 경우 | 문구에 충돌 이유를 명시하는 방향으로 보완 가능 |
| 동일 섹터 내 ETF와 개별주가 동시에 후보인 경우 | 이번 주 시장 분위기에 더 적합한 쪽을 선택. admin이 최종 판단. |
| 예외 연속 추천 사유가 있으나 확신하기 어려운 경우 | 보류 처리 후 1~2일 내 추가 확인 |
| 가격 구간이 약간 벗어났으나 방향성은 맞는 경우 | 수치 업데이트 후 승인 허용 |

---

## 12. 확정 현황

### 확정된 운영 원칙

| 항목 | 확정 내용 |
|------|----------|
| 검수 단계 순서 | 정책 준수 → 로직 일관성 → 개별 종목 → ETF 추가 → 문구 표현 → 최종 결정 |
| 섹터 중복 즉시 반려 | 정책 위반으로 즉시 처리 |
| 3주 연속 추천 즉시 반려 | 예외 없음 |
| 리스크 누락 처리 | 2개 이상 누락 시 반려. 1개 누락 시 수정 요청 |
| ETF 환헤지/레버리지 누락 처리 | 반려 수준 |
| 예외 연속 추천 기록 의무 | 승인 시 검수 메모 기록 필수 |
| 금지 표현 유형 | 확신 표현, 수익률 숫자, 즉각 매수 촉구, 단정적 방향 예측 |

### 정책 확정이 더 필요한 항목

| 항목 | 미확정 이유 | 확정 필요 시점 |
|------|-----------|--------------|
