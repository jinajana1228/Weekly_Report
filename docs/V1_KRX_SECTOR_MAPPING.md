# V1 KRX 업종 → 서비스 섹터 매핑

> **문서 목적**: KRX(한국거래소) 업종 분류 체계를 이 서비스의 13개 섹터 코드로 매핑하는 규칙을 정의한다.
> 이 매핑 테이블은 favored/cautious_sectors 판단, 메인 5개 섹터 중복 검사, 추천 카드 섹터 표시에 사용된다.
> 이 문서는 설계 문서이며, 실제 데이터 연동·코드 구현은 다음 단계에서 진행한다.

---

## 1. 서비스 13개 섹터 코드 정의

| 섹터 코드 | 의미 | 대표 산업 |
|---------|------|---------|
| `TECH` | 기술·IT·반도체·소프트웨어 | 반도체, 디스플레이, IT 서비스, 게임, 플랫폼 |
| `BATTERY` | 배터리·이차전지·전기차 밸류체인 | 배터리 셀, 양극재/음극재/전해질/분리막 |
| `HEALTHCARE` | 헬스케어·바이오·의료기기 | 제약, 바이오, 의료기기, 의료정밀 |
| `FINANCE` | 금융 전 업종 | 은행, 보험, 증권, 금융지주 |
| `CONSUMER` | 소비재·유통·생활 | 식품, 화장품, 패션, 유통, 미디어·엔터 |
| `INDUSTRIAL` | 산업재·기계·운송 | 자동차, 조선, 항공, 일반기계, 방산 |
| `MATERIAL` | 소재·화학·철강 | 석유화학, 철강, 비금속광물, 종이목재 |
| `ENERGY` | 에너지·유틸리티 | 전기, 가스, 정유, 신재생에너지 |
| `TELECOM` | 통신 | 이동통신, 유선통신, 통신인프라 |
| `REALESTATE` | 부동산·건설 | 건설, 시행, REITs |
| `ETF_DOMESTIC` | 국내 주식 추종 ETF | KOSPI200, 코스닥150, KRX 업종 ETF |
| `ETF_OVERSEAS` | 해외 지수 추종 ETF | S&P500, NASDAQ, 중국·일본 등 해외 지수 ETF |
| `ETF_BOND_DIV` | 채권형·배당형·인컴형 ETF | 국내 채권 ETF, 배당 ETF, 혼합 인컴 ETF |

## 2. KRX 업종 분류 체계 개요

KRX는 KOSPI 상장 종목에 대해 아래 업종 대분류를 사용한다. 이 문서에서는 KOSPI 기준 주요 업종지수 항목을 기준으로 한다. KOSDAQ도 유사한 분류를 사용하나 세부 구성이 다를 수 있다.

| KRX 업종명 | 설명 |
|---------|------|
| 전기전자 | 반도체, 디스플레이, 통신기기, 전자부품 |
| 화학 | 석유화학, 고무, 플라스틱, 화학소재 |
| 의약품 | 제약, 바이오의약품 |
| 의료정밀 | 의료기기, 광학기기, 정밀기계 |
| 금융업 | 은행, 보험, 증권, 금융지주 |
| 서비스업 | IT서비스, 소프트웨어, 게임, 플랫폼, 미디어, 숙박·여행 |
| 운수장비 | 자동차, 조선, 항공기 |
| 기계 | 일반기계, 방산, 엔지니어링 |
| 음식료품 | 식품, 음료, 담배 |
| 유통업 | 도소매, 이커머스 |
| 섬유의복 | 섬유, 의류, 화장품 일부 |
| 철강금속 | 철강, 비철금속 |
| 비금속광물 | 시멘트, 유리, 세라믹 |
| 종이목재 | 제지, 목재 |
| 건설업 | 종합건설, 전문건설, 엔지니어링 |
| 전기가스업 | 전력, 가스, 수도 |
| 통신업 | 이동통신, 유선통신 |
| 운수창고업 | 물류, 항공운수, 해운 |

> **주의**: KRX 업종 분류와 이 서비스의 13개 섹터 코드는 1:1 대응이 되지 않는다. 일부 KRX 업종은 여러 서비스 섹터로 나뉘거나, 서비스 섹터 하나에 여러 KRX 업종이 묶인다. 아래 섹션에서 매핑 방식을 상세히 정의한다.

## 3. 매핑 방식 분류

| 매핑 유형 | 설명 | 예시 |
|---------|------|------|
| **직접 매핑** | KRX 업종 1개 → 서비스 섹터 1개. 해석 없이 자동 적용 가능 | 통신업 → TELECOM, 의약품 → HEALTHCARE 일부 |
| **묶음 매핑** | KRX 업종 여러 개 → 서비스 섹터 1개 | 음식료품 + 유통업 + 섬유의복 → CONSUMER |
| **분리 매핑** | KRX 업종 1개 → 서비스 섹터 여러 개 중 하나로 분류 | 서비스업 → TECH 또는 CONSUMER (종목별 판단) |
| **운영 판단 필요** | KRX 업종만으로 섹터 결정이 어려운 경우. 운영자 직접 분류 | 화학 내 배터리 소재 기업 → BATTERY 또는 MATERIAL |
| **ETF 별도 규칙** | KRX 업종 분류 적용 불가. 추종 지수·운용 방식 기반으로 분류 | ETF 전 종목 → ETF_DOMESTIC / ETF_OVERSEAS / ETF_BOND_DIV |

## 4. 섹터별 상세 매핑

### TECH

| 항목 | 내용 |
|-----|------|
| **포함되는 KRX 업종** | 전기전자 (반도체·디스플레이·전자부품 위주), 서비스업 (IT서비스·소프트웨어·게임·플랫폼) |
| **매핑 유형** | 묶음 매핑 + 분리 매핑 (서비스업은 분리 적용) |
| **대표 종목군** | 반도체 제조, 디스플레이 패널, 통신기기, IT 서비스, 게임 개발사, 온라인 플랫폼 |
| **경계 사례** | ① 배터리 셀·소재 기업은 전기전자로 분류되지만 BATTERY로 분리 <br> ② 서비스업 중 비IT 소비재 서비스(숙박·여행 등)는 CONSUMER로 분리 <br> ③ 통신장비 제조사는 전기전자에 속하나 TECH 유지 (TELECOM 아님) |
| **운영 비고** | 서비스업 내 종목은 사업 내용 기반 TECH vs CONSUMER 판단 필요. 게임/SW/플랫폼 → TECH, 미디어/엔터/여행 → CONSUMER |

---

### BATTERY

| 항목 | 내용 |
|-----|------|
| **포함되는 KRX 업종** | 전기전자 일부 (배터리 셀 제조사), 화학 일부 (배터리 소재: 양극재·음극재·전해질·분리막) |
| **매핑 유형** | 운영 판단 필요 (KRX 업종 단독으로 결정 불가) |
| **대표 종목군** | 배터리 셀 제조, 양극재 제조, 음극재 제조, 전해질, 분리막, 배터리 팩·모듈 |
| **경계 사례** | ① 동일 기업이 화학과 배터리 소재를 모두 하는 경우 주 매출 비중 기준으로 판단 <br> ② 전기차 제조사 자체는 INDUSTRIAL로 분류. 배터리 부품 전담 자회사만 BATTERY |
| **운영 비고** | BATTERY는 KRX 공식 업종 코드가 아니다. 서비스 운영자가 유니버스 구성 시 종목별로 직접 BATTERY로 분류 지정해야 한다. 자동화 불가. |

**BATTERY가 독립 섹터인 이유**:
KRX 업종 체계에서 배터리 관련 기업은 전기전자(셀 제조사) 또는 화학(소재)으로 분산 분류된다. 그러나 이차전지·전기차 밸류체인은 시장에서 하나의 독립 테마로 움직이며, 글로벌 EV 전환 정책과 연동된 모멘텀이 다른 섹터와 구분된다. 이 서비스는 1개월 관점 추천이므로 테마 모멘텀 방향성이 중요하다. 배터리를 TECH 또는 MATERIAL에 합쳐 넣으면 ① 섹터 방향성 분석이 왜곡되고, ② favored_sectors에서 배터리 테마가 다른 섹터에 묻혀 추천이 누락된다. 따라서 BATTERY를 독립 섹터로 분리하는 것이 서비스 운영에 적합하다.

### TELECOM

| 항목 | 내용 |

## 5. ETF 섹터의 특수성 — KRX 업종 체계 적용 불가

**ETF_DOMESTIC, ETF_OVERSEAS, ETF_BOND_DIV 3개 섹터는 KRX 업종 분류 체계로 결정할 수 없다.**

KRX는 ETF 종목에 대해 업종 코드를 일반 주식처럼 부여하지 않는다. 대신 ETF는 아래 기준으로 분류한다:

| 분류 기준 | 판단 방법 |
|---------|---------|
| 추종 지수의 지역 | 국내 지수 → ETF_DOMESTIC, 해외 지수 → ETF_OVERSEAS |
| 자산 유형 | 주식 지수 → ETF_DOMESTIC/ETF_OVERSEAS, 채권 또는 배당 → ETF_BOND_DIV |
| 레버리지/인버스 여부 | 별도 플래그 부여. 소속 섹터는 기초지수 지역 기준으로 결정 |
| 혼합형 | 주 구성 자산 비중 기준으로 결정 |

ETF 섹터 분류는 **ETF 메타 데이터 (추종지수, 자산유형) 기반의 운영자 직접 분류**가 원칙이다. KRX 업종 코드나 자동화된 업종 매핑으로는 처리할 수 없다.

## 6. 예외 분류 규칙

### 6-1. 다중 업종 노출 종목 처리

한 종목이 여러 사업 부문을 보유하여 서비스 섹터가 모호한 경우:

| 판단 원칙 | 적용 방법 |
|---------|---------|
| 주 매출 비중 기준 | 가장 큰 매출 세그먼트의 사업 성격으로 섹터 결정 |
| 향후 성장 방향 기준 | 현재 매출보다 성장 모멘텀이 집중된 사업 방향으로 결정 가능 (운영자 판단) |
| 시장 인식 기준 | 주식 시장에서 해당 종목을 어떤 테마로 묶어서 거래하는지 참조 |
| 분기별 점검 | 사업 구조 변화 시 섹터 재분류 가능 |

**예시**:
- 삼성SDI: 배터리 셀(BATTERY) + 전자재료(MATERIAL) 겸업 → 배터리 부문 매출 비중 및 모멘텀 기준으로 BATTERY 분류
- LG화학: 배터리 소재 + 일반화학 → 배터리 분사 후 남은 법인은 MATERIAL, 분사 자회사는 BATTERY

### 6-2. 서비스 섹터와 경계가 모호한 주요 사례

| 종목 유형 | 경계 상황 | 권장 분류 |
|---------|---------|---------|
| 핀테크 기업 (금융 라이선스 보유) | TECH vs FINANCE | FINANCE |
| 핀테크 기업 (소프트웨어 전문, 라이선스 없음) | TECH vs FINANCE | TECH |
| 전기차 완성차 | INDUSTRIAL vs BATTERY | INDUSTRIAL (배터리 공급사는 BATTERY) |
| 배터리 소재 겸업 화학사 | MATERIAL vs BATTERY | 주력 매출 비중 기준 운영자 판단 |
| 정유사 (화학 분류) | MATERIAL vs ENERGY | ENERGY (에너지 공급이 본업) |
| 신재생에너지 겸업 화학사 | MATERIAL vs ENERGY | 운영자 판단 |
| IT 서비스 기반 의료 플랫폼 | TECH vs HEALTHCARE | HEALTHCARE (헬스케어가 최종 서비스 대상) |
| 방산·우주 기업 | INDUSTRIAL (기계/항공) | INDUSTRIAL |
| 대형 건설·엔지니어링 | REALESTATE vs INDUSTRIAL | 주력 공사 유형 기준 운영자 판단 |

### 6-3. 경계 사례 처리 원칙

1. **판단 문서화**: 운영자가 경계 사례를 BATTERY/TECH 등 특정 섹터로 분류할 때, 유니버스 파일에 해당 종목의 섹터와 분류 사유를 함께 기록한다.
2. **분기별 점검**: 사업 구조가 변화하는 종목(배터리 분사, 신사업 확장 등)은 분기별 유니버스 갱신 시 재검토한다.
3. **이중 분류 금지**: 하나의 종목은 하나의 서비스 섹터에만 속한다. 이중 분류는 섹터 중복 검사 오류를 유발한다.

## 7. 매핑 테이블이 서비스 운영에 쓰이는 방식

### 7-1. favored_sectors / cautious_sectors 판단

1. KRX 업종지수 주간 등락률·수급 방향 데이터를 수집한다.
2. 이 문서의 매핑 테이블을 통해 KRX 업종 → 서비스 섹터 코드로 변환한다.
3. 변환된 섹터 코드별 방향성 수치를 확인하고, 운영자가 favored/cautious 섹터를 판단하여 입력한다.

> **주의**: BATTERY, ETF 3개 섹터는 KRX 업종지수로 직접 확인이 어렵다. BATTERY 관련 방향성은 배터리 관련 종목들의 개별 수급 방향과 글로벌 EV 지수를 보조 신호로 활용한다.

### 7-2. 메인 5개 추천 섹터 중복 검사

1. 후보 종목 각각의 서비스 섹터 코드를 조회한다 (유니버스 파일에 사전 등록).
2. 최종 5개 선정 시 동일 섹터 코드가 2개 이상 포함되지 않도록 확인한다.
3. ETF 3개 섹터(ETF_DOMESTIC, ETF_OVERSEAS, ETF_BOND_DIV)는 개별주 섹터와 독립적으로 취급한다.

### 7-3. 추천 카드 섹터 배지 표시

1. 종목 또는 ETF의 서비스 섹터 코드를 추천 카드에 배지로 표시한다.
2. 표시 레이블은 서비스 섹터 코드(TECH, BATTERY 등)를 그대로 사용하거나, UI에서 한글 레이블을 매핑하여 표시한다.
3. 섹터 코드가 favored_sectors에 포함된 경우 카드에 추가 시각적 표시를 할 수 있다 (구현 단계에서 결정).

### 7-4. same_sector_alternatives 선택

1. 메인 추천 종목의 서비스 섹터 코드와 동일한 섹터의 종목을 유니버스에서 선별한다.
2. alternatives 배제 원칙(V1_RECOMMENDATION_POLICY_FINALIZATION.md 3-1절): 직전 주 메인 추천 종목은 원칙적으로 alternatives에서도 배제한다.
3. ETF 섹터는 ETF끼리만 alternatives로 구성하거나, 개별주 섹터의 alternatives로 ETF를 추가하는 것도 허용된다 (동일 섹터 커버리지 ETF에 한해).

- 서비스 13개 섹터 코드 정의 및 의미
