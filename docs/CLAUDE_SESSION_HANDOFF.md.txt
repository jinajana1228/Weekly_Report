---

## 현재까지 확정된 운영 원칙
1. V1은 자동 수집 중심
2. 사람은 admin에서 최종 발행 승인 여부만 판단
3. 뉴스는 수치 데이터의 대체재가 아니라 보완 신호
4. 뉴스 부족 시 수치 기반 발행 가능
5. approval.json은 에디션 최종 발행 게이트
6. signal_review.json은 뉴스 신호 검수 결과 파일
7. V1은 파일 기반 운영
8. archive detail 충돌은 V1에서 덮어쓰기 허용

---

## 서비스 구조
- 국내 주식 및 ETF 대상
- 메인 추천 5개
- 메인 추천 5개는 섹터 중복 없음
- 각 메인 카드에 same_sector_alternatives 2개
- ETF는 독립 리포트 단위
- 주간 발행형
- 1개월(2~4주) 관점 추천
- Admin 검수 후 발행
- draft / current / archive 운영

## 현재 구현 완료 상태
- UI read-only 구현 완료

- admin 화면 read-only 구현 완료
- /admin 보호 구현 완료
- local publish dry-run 스크립트 구현 완료

## 현재까지 주요 결정
- approval = 발행 게이트
- signal_review = 참고 입력, 비차단
- 뉴스는 linked_signal_ids[] 중심 참조
- read-only 단계에서는 write 기능 없음
- /admin 보호는 middleware + secret token 방식으로 구현 완료
- local publish dry-run은 차단조건/경고조건을 구분하여 구현 완료
