# V1 fallback 운영 정책 초안

> **문서 목적**: admin 검수 지연, 반려, 승인 실패 등 비정상 운영 상황에서의 처리 원칙을 정의한다.
> **중요**: 이 문서는 초안이다. 세부 승인자 체계, 대체 승인자 지정, 알림 방식 등은 미확정이며 후속 결정이 필요하다.

---

## fallback 정책의 핵심 원칙

> **기존 current는 명시적 승인 없이 절대 변경되지 않는다.**

어떤 상황에서도 admin의 승인 없이 draft가 current로 전환되는 경우는 없다.
따라서 모든 fallback 상황에서 기본 동작은 **기존 current 유지**이다.

---

## 상황별 처리 정책

### 상황 1: 월요일 오전에 admin 검수가 시작되지 않은 경우

| 항목 | 내용 |
|------|------|
| public 화면 | 기존 current 에디션 유지 (자동) |
| draft 상태 | `UNDER_REVIEW` 또는 `DRAFT` 유지 |
| 권장 조치 | 가능한 빠른 시간 내 검수 시작 |
| 공지 필요 여부 | V1 기준 별도 공지 없음 (기존 에디션이 계속 표시되므로 사용자에게는 변화 없음) |
| 최대 허용 지연 시간 | **미확정** (후속 정책 결정 필요) |

---

### 상황 2: 검수 중 반려 (REJECTED) 후 재준비가 필요한 경우

| 항목 | 내용 |
|------|------|
| public 화면 | 기존 current 유지 |
| draft 상태 | `REJECTED` 유지, 수정 대기 |
| 권장 조치 | 반려 사유(reviewer_note) 확인 후 draft 수정 → 재검수 요청 |
| 재검수 시 | approval.json 초기화 후 검수 흐름 재시작 |
| 수정 마감 기준 | **미확정** (이번 주 안에 재발행할지, 다음 주로 미룰지 기준 없음) |

---

### 상황 3: 보류 (HOLD) 상태가 장기화되는 경우

| 항목 | 내용 |
|------|------|
| public 화면 | 기존 current 유지 |
| draft 상태 | `HOLD` 유지 |
| 권장 조치 | 보류 사유 재검토 후 승인 또는 반려로 전환 |
| 보류 장기화 기준 | **미확정** |
| 사용자 공지 필요 여부 | **미확정** |

---

### 상황 4: 전환 프로세스 실행 중 오류 발생

| 항목 | 내용 |
|------|------|
| 발생 시점 | 승인 후 draft → current 전환 스크립트 실행 중 오류 |
| 원칙 | 부분 전환 상태를 허용하지 않는다. 전환 작업 전체를 중단하고 이전 상태 유지 |
| data/current/ | 변경 전 상태로 복원 |
| data/archive/ | 불완전하게 이동된 경우 수동 복구 필요 |
| 권장 조치 | 오류 원인 파악 후 수동 또는 재실행 |
| 자동 롤백 여부 | **미확정** (구현 단계에서 결정) |

---

### 상황 5: Git push 실패로 Vercel 반영이 되지 않은 경우

| 항목 | 내용 |
|------|------|
| 발생 시점 | 파일 전환은 완료됐으나 Git push 실패 |
| public 화면 | Vercel이 기존 커밋을 바라보므로 기존 에디션 유지 |
| 권장 조치 | Git push 재시도 후 Vercel 재배포 확인 |
| 자동 재시도 여부 | **미확정** |

---

### 상황 6: 이번 주 에디션을 발행하지 않기로 결정한 경우

| 항목 | 내용 |
|------|------|
| 가능 여부 | 가능. 이번 주 발행 건너뜀 |
| public 화면 | 기존 current 유지 (사용자에게는 이전 에디션이 계속 표시됨) |
| draft 처리 | 신규 draft 생성 없이 유지, 또는 폐기 |
| 아카이브 영향 | 발행 이력에 공백이 생기므로 archive 폴더에 해당 주차 없음 |
| 공지 필요 여부 | **미확정** |
| 주차 공백 허용 여부 | V1에서는 허용. 단, archive 목록 UI에서 공백 주차를 어떻게 표시할지는 구현 단계 결정 |

---

## 후속 운영자가 확인해야 할 포인트

매주 운영 시 아래 사항을 체크해야 한다.

1. 일요일 밤 draft 생성이 완료되었는지 (`data/draft/manifest.json` 존재 여부 확인)
2. `admin/approval.json`의 `decision` = `PENDING` 상태인지 (검수 대기 확인)
3. draft의 `picks` 폴더에 파일 5개가 모두 있는지
4. 월요일 오전 검수 완료 및 current 전환이 반영되었는지 (Vercel 화면 확인)
5. Git에 `data/current/`, `data/archive/` 변경이 정상 커밋/푸시되었는지

| 발행 건너뜀 시 사용자 공지 여부 | 미확정 |
| 전환 실패 시 자동 롤백 방식 | 미확정 |
| Git push 실패 시 재시도 방식 | 미확정 |
| archive 오류 수정 정책 (errata) | 미확정 |
