# V1 데이터 소스 Shortlist

> **문서 목적**: V1 서비스 운영에 필요한 데이터 영역별로 현실적인 소스 후보군을 좁히고, 권장 shortlist와 후순위 후보를 구분한다.
> 이 문서는 설계 문서이며, 실제 소스 확정·API 연동·코드 구현은 다음 단계에서 진행한다.
> "권장 shortlist"는 우선 검토 순서를 제안하는 것이며, 최종 확정이 아니다.

---

## 1. 소스 층위 체계

| 층위 | 의미 | 신뢰도 |
|-----|------|-------|
| **공식** | 정부기관·거래소·금융당국이 직접 발행하는 데이터. 오류 시 소스에서 수정 가능. | 최상 |
| **준공식** | 공공 성격의 민간 기관 또는 증권사가 제공하는 데이터. 공식 소스 기반이나 제공 방식은 민간. | 높음 |
| **보조** | 비공식 채널이지만 데이터 품질이 준수하고 접근이 용이한 소스. 공식 소스로 교차 확인 권장. | 보통 |
| **수동 입력** | 자동 수집이 부적합하거나 운영자 판단이 필요한 데이터. 운영자가 직접 구조화하여 입력. | 판단 의존 |

---

## 2. 데이터 영역별 Shortlist

### 2-1. 국내 시세 / 거래대금

**필요한 이유**: 주가(동전주 필터·가격 위치 평가), 거래대금(저유동성 필터), 등락률(급등 과열 필터)은 Hard Filter와 추천 평가의 핵심 입력값이다. 이 데이터 없이 추천 엔진이 실행될 수 없다.

**사용 위치**: Hard Filter 전 항목(동전주/저유동성/급등), 가격 위치 평가, 52주 고저, 이동평균 계산

| 소스 유형 | 소스 후보 | 층위 | 자동 수집 적합 | 권장 우선순위 |
|---------|---------|------|------------|------------|
| 거래소 공식 시스템 | KRX 정보데이터시스템 (data.krx.co.kr) | 공식 | ✅ 적합 | **1순위** |
| 증권사 OpenAPI | KIS Developers (한국투자증권), 키움 OpenAPI+ 등 | 준공식 | ✅ 적합 | **2순위** |
| 금융 정보 포털 | 네이버 금융, 다음 금융 | 보조 | ⚠️ 비공식, 구조 변경 위험 | 후순위 |

**권장 shortlist**: KRX 정보데이터시스템 (공식, 무료) → 연동 어려울 경우 증권사 OpenAPI (KIS 또는 키움)

**수동 보완 필요 여부**: 불필요. 자동 수집이 충분히 가능한 영역.

**최종 확정 전 검토 포인트**:
- KRX 정보데이터시스템의 API 호출 한도 및 갱신 주기 확인 필요
- 증권사 OpenAPI는 계좌 개설 등 가입 절차 필요 여부 확인 필요
- 일간 종가 기준인지 실시간 기준인지 서비스 요건에 맞게 결정 필요 (V1은 일간 종가 기준으로 충분)

---

### 2-2. 투자자 수급 (외국인 / 기관 / 개인)

**필요한 이유**: 외국인·기관의 5일/20일 순매수는 수급 신호 평가의 핵심 입력값이다. 국내 시장 요약의 수급 방향 판단에도 사용한다.

**사용 위치**: 수급 신호 평가 (개별 종목), 국내 시장 요약 (KOSPI 전체 수급)

| 소스 유형 | 소스 후보 | 층위 | 자동 수집 적합 | 권장 우선순위 |
|---------|---------|------|------------|------------|
| 거래소 공식 시스템 | KRX 정보데이터시스템 (투자자별 매매현황) | 공식 | ✅ 적합 | **1순위** |
| 금융투자협회 | KOFIA 통계 포털 (kofia.or.kr) | 준공식 | ✅ 가능 (주간 단위) | **2순위** |
| 증권사 OpenAPI | KIS, 키움 등 투자자별 순매수 데이터 | 준공식 | ✅ 적합 | **2순위 (병행)** |

**권장 shortlist**: KRX 정보데이터시스템 (종목별 투자자별 매매현황)

**수동 보완 필요 여부**: 불필요. 시장 전체 수급 요약은 수치 취합 후 시장 요약 문구 작성 시 운영자가 해석 입력.

**최종 확정 전 검토 포인트**:
- KRX 종목별 외국인/기관 수급은 T+1 기준(전일 마감 후 공개)인지 확인 필요
- 5일/20일 롤링 합산 계산이 가능한 데이터 형태인지 확인 필요

---

### 2-3. 지수 / 업종지수 / 상대강도 계산용 데이터

**필요한 이유**: KOSPI/KOSDAQ 주간 등락률은 국내 시장 요약의 핵심이다. KOSPI 업종지수 주간 등락률은 favored/cautious_sectors 판단의 주된 수치 근거다. 상대강도 계산은 업종 지수 시계열이 필요하다.

**사용 위치**: 국내 시장 요약, favored_sectors / cautious_sectors 판단, 시장 적합도 평가

| 소스 유형 | 소스 후보 | 층위 | 자동 수집 적합 | 권장 우선순위 |
|---------|---------|------|------------|------------|
| 거래소 공식 시스템 | KRX 정보데이터시스템 (업종지수 시세) | 공식 | ✅ 적합 | **1순위** |
| 증권사 OpenAPI | KIS, 키움 업종지수 | 준공식 | ✅ 적합 | **2순위** |
| 금융 정보 포털 | 네이버 금융 업종 페이지 | 보조 | ⚠️ 비공식 | 후순위 |

**권장 shortlist**: KRX 정보데이터시스템 → 업종지수 일간 시세 데이터

**수동 보완 필요 여부**:
- 수치 자체는 자동 수집 가능
- **KRX 업종 → 서비스 13개 섹터 매핑**은 수동 설정 후 고정 테이블로 관리 (별도 매핑 문서 참조)
- favored/cautious 섹터 최종 지정은 수치 확인 후 운영자 판단 입력

**최종 확정 전 검토 포인트**:
- KRX 업종지수는 코스피 기준 약 18개 대분류. 이를 서비스 13개 섹터로 변환하는 매핑 테이블 사전 구축 필요
- 4주·8주 상대강도 계산을 위해 업종지수 히스토리 데이터 최소 2개월치 필요

---

### 2-4. 거래소 지정 / 경고 / 정지 현황

**필요한 이유**: 관리종목·거래정지·투자경고·투자위험·단기과열·투자주의 지정 여부는 Hard Filter의 1순위 항목이다. 이 데이터 없이 Hard Block이 작동하지 않는다. 또한 발행 직전에 재확인이 필수다 (검수 후 상태 변경 가능).

**사용 위치**: Hard Filter 2-1절 전체, 발행 직전 체크리스트

| 소스 유형 | 소스 후보 | 층위 | 자동 수집 적합 | 권장 우선순위 |
|---------|---------|------|------------|------------|
| 거래소 공식 시스템 | KRX 시장감시 / KRX 정보데이터시스템 (투자유의 지정 현황) | 공식 | ✅ 적합 | **1순위** |
| 거래소 공시 게시판 | KRX 공시 (투자유의·관리·경고 지정 게시판) | 공식 | ⚠️ 공시 게시판 파싱 필요 | **보조** |
| 증권사 OpenAPI | 일부 증권사 API에서 투자유의 여부 제공 | 준공식 | ✅ 일부 가능 | **2순위 (확인 필요)** |

**권장 shortlist**: KRX 공식 투자유의 종목 목록 (KRX 정보데이터시스템 또는 시장감시 데이터)

> **중요**: 거래소 지정 현황은 실시간·당일 기준으로 바뀔 수 있다. V1에서는 draft 작성 시 1회 + 발행 직전 1회 총 2회 확인이 원칙이다 (V1_HARD_FILTER_POLICY.md 5절).

**수동 보완 필요 여부**: 자동 수집 가능하나, 발행 직전 재확인은 운영자가 수동으로 최종 점검하는 것을 권장. 완전 자동화에만 의존하면 지정 상태 변경을 놓칠 수 있다.

**최종 확정 전 검토 포인트**:
- KRX 정보데이터시스템에서 관리/경고/위험/주의/단기과열/거래정지를 단일 API로 조회 가능한지 확인 필요
- 일부 항목은 별도 데이터셋으로 분리되어 있을 수 있어 조회 방식 통합 설계 필요

---

### 2-5. 재무 데이터 / 공시

**필요한 이유**:
- **재무 데이터**: TTM 영업이익(적자 필터), 부채비율(재무 안전성 평가), 매출 추이 등
- **공시 데이터**: 최근 주요 공시 요약(단기 촉매 평가), 실적 발표 일정, 어닝 서프라이즈 판단

**사용 위치**: Hard Filter 2-2절(적자), 재무 안전성 평가, 단기 촉매 평가

| 소스 유형 | 소스 후보 | 층위 | 자동 수집 적합 | 권장 우선순위 |
|---------|---------|------|------------|------------|
| 금감원 공시 시스템 | DART (dart.fss.or.kr) — 사업보고서/분기보고서 | 공식 | ✅ API 제공 | **1순위 (재무)** |
| 금감원 공시 시스템 | DART — 공시 목록 조회 (최근 공시) | 공식 | ✅ API 제공 | **1순위 (공시)** |
| 거래소/증권사 | 실적 발표 일정 데이터 | 준공식 | ✅ 가능 (일부) | **2순위** |
| 금융 정보사 | 에프앤가이드 (FnGuide), 에프앤데이터 | 준공식 | ✅ 가능 (유료 가능성) | **후순위 (비용 확인 후)** |

**권장 shortlist**:
- 재무 데이터: **DART OpenAPI** (사업보고서, 분기보고서 기반 재무제표)
- 공시 목록: **DART OpenAPI** (최근 공시 목록 조회)

**수동 보완 필요 여부**:
- 재무 수치 자체는 DART에서 자동 수집 가능. 단, 분기 보고서 기준이므로 갱신은 분기 1회.
- 공시 목록은 자동 수집 가능하나, "주요 공시인지 여부"의 판단(계약·인허가 등 중요도 분류)은 운영자 수동 검토 필요.
- 어닝 서프라이즈 여부 판단(컨센서스 대비 비교)은 컨센서스 데이터가 별도로 필요하므로, V1에서는 운영자 수동 판단 권장.

**최종 확정 전 검토 포인트**:
- DART OpenAPI 일 호출 한도 확인 필요 (유니버스 종목 수 × 분기 보고서 건수 고려)
- TTM 영업이익 계산을 위해 4개 분기 합산 로직 별도 구현 필요 (단순 API 응답값이 아닐 수 있음)
- 컨센서스 데이터(어닝 서프라이즈 판단용)는 별도 유료 데이터 소스 필요 여부 판단 필요

### 2-6. ETF 메타 데이터

**필요한 이유**: 추종 지수, 환헤지 여부, 레버리지 타입, TER, AUM, 구성 종목은 ETF 상세 리포트의 필수 항목이다. 레버리지 타입은 급등 과열 필터 환산 계산에도 필요하다.

**사용 위치**: ETF 상세 리포트, Hard Filter 2-6절(레버리지 ETF 급등 환산), Admin ETF 설명 검수

| 소스 유형 | 소스 후보 | 층위 | 자동 수집 적합 | 권장 우선순위 |
|---------|---------|------|------------|------------|
| 금융투자협회 | KOFIA 펀드 공시 (펀드정보 포털, dis.kofia.or.kr) | 준공식 | ✅ 일부 가능 | **1순위** |
| KRX ETF 섹션 | KRX 정보데이터시스템 ETF 메타 | 공식 | ✅ 가능 | **1순위 (병행)** |
| 운용사 공식 홈페이지 | 각 운용사 ETF 상세 페이지 (삼성자산운용, 미래에셋 등) | 공식 | ⚠️ 형식 다양, 수동 병행 | **보조** |
| 네이버 금융 ETF | ETF 개요 정보 | 보조 | ⚠️ 비공식 | 후순위 |

**권장 shortlist**:
- KRX 정보데이터시스템 (ETF 기본 정보: AUM, TER, 기초지수명)
- KOFIA 펀드 공시 (구성 종목, 환헤지 여부)
- 운용사 공시 페이지 (레버리지 배율 등 제품 특성)

**수동 보완 필요 여부**:
- AUM, TER, 추종지수는 자동 수집 가능. 단, 각 소스마다 필드 형식이 다르므로 파싱 규칙 별도 설계 필요.
- 국가/지역 노출 비중, 섹터 노출 비중은 운용사별 공시 형식이 달라 수동 확인 후 입력이 현실적.
- **ETF 메타는 변경 빈도가 낮으므로 V1에서는 월간~분기 수동 갱신도 허용 가능.**

**최종 확정 전 검토 포인트**:
- KRX ETF 메타 API에서 레버리지 배율 정보가 제공되는지 확인 필요
- KOFIA와 KRX의 ETF 정보 중 어느 소스가 더 최신·완전한지 비교 확인 필요
- 해외 ETF (ETF_OVERSEAS 분류 종목)의 경우 국내 상장 형태이므로 동일 소스로 처리 가능

### 2-7. 글로벌 지수 / 금리 / 환율

**필요한 이유**:
- **글로벌 지수 (S&P 500, NASDAQ, VIX)**: 글로벌 시장 방향·변동성 판단. ETF 우선 추천 트리거.
- **미국 10년 국채 금리**: 성장주 vs 가치주 방향, 금리 환경 판단.
- **USD/KRW 환율**: 원화 강약 판단, 수출 업종 방향성.

**사용 위치**: 글로벌 시장 요약, favored_sectors 판단, ETF 추천 판단, 국내 시장 요약

| 소스 유형 | 소스 후보 | 층위 | 자동 수집 적합 | 권장 우선순위 |
|---------|---------|------|------------|------------|
| 미국 연준 공식 통계 | FRED (Federal Reserve Economic Data, fred.stlouisfed.org) — 금리, DXY | 공식 (미국) | ✅ API 제공 (무료) | **1순위 (금리/DXY)** |
| 한국은행 공식 통계 | 한국은행 ECOS (ecos.bok.or.kr) — 환율, 한국 금리 | 공식 (국내) | ✅ API 제공 (무료) | **1순위 (환율/국내 금리)** |
| 글로벌 금융 데이터 | Yahoo Finance (finance.yahoo.com) — 지수, VIX | 보조 | ✅ 비공식 API 존재 | **2순위 (지수/VIX)** |
| 글로벌 금융 데이터 | Alpha Vantage — 지수, 환율 | 보조 | ✅ API 제공 (무료/유료 구분) | **2순위 (대안)** |
| 거래소/유료 데이터 | Bloomberg, Refinitiv (Thomson Reuters) | 준공식 (유료) | ✅ 고품질 | 후순위 (비용 부담) |

**권장 shortlist**:
- 환율: **한국은행 ECOS** (USD/KRW 공식 기준)
- 미국 금리: **FRED** (미국 10년 국채 금리, FFR)
- 글로벌 지수·VIX: **Yahoo Finance** (비공식이나 접근성 높음, 공식 소스로 교차 확인 권장)

**수동 보완 필요 여부**:
- 수치 자체는 자동 수집 가능.
- FOMC 스탠스(매파/비둘기파 판단)는 수치 데이터로 자동화 불가. 연준 공식 성명 발표 후 운영자가 스탠스 방향 입력.
- 글로벌 시장 요약 문구는 수치 확인 후 운영자 작성.

**최종 확정 전 검토 포인트**:
- Yahoo Finance 비공식 API는 서비스 정책 변경으로 접근이 막힐 수 있음. 안정성 우선 시 유료 소스 검토 필요.
- FRED API 무료 사용 시 일 호출 한도(500 req/day) 확인 필요.
- 한국은행 ECOS API 가입 및 키 발급 절차 확인 필요.

### 2-8. 뉴스 신호 입력 방식

**필요한 이유**: 수치 데이터만으로 판단이 어려운 정책 방향성, 업황 변화 신호, 개별 종목 이슈를 보완한다. 기사 전문 재배포는 금지이므로 구조화된 방향 신호만 활용한다.

**사용 위치**: favored/cautious_sectors 판단 보완, 단기 촉매 평가, Admin 리스크 검수

| 소스 유형 | 소스 후보 | 층위 | 자동 수집 적합 | 권장 우선순위 |
|---------|---------|------|------------|------------|
| 수동 구조화 입력 | 운영자가 주요 뉴스 모니터링 후 신호 방향만 입력 | 수동 입력 | ❌ 자동화 부적합 | **V1 권장 방식** |
| 공시 연계 자동 | DART 공시 목록 (종목별 최근 공시 유형 확인) | 공식 | ✅ 적합 (공시는 공식) | **공시 연계만 자동화** |
| 뉴스 API | 네이버 뉴스 API, 빅카인즈 등 | 보조 | ⚠️ 기사 전문 저장 금지 규칙 적용 | 후순위 |

**V1 권장 뉴스 신호 처리 방식**:

뉴스 신호는 완전 자동화 대신 아래 구조화 입력 방식을 권장한다.

```
입력 형식 (예시 개념):
{
  "sector": "TECH",
  "direction": "bullish",
  "catalyst_type": "정책",
  "summary": "반도체 국가전략 기술 추가 지정 검토",
  "confidence": "medium"
}
```

운영자가 주간 draft 작성 전 뉴스를 모니터링하고, 위 형식으로 섹터·방향·촉매유형·신뢰도만 입력한다. 기사 원문은 저장하지 않는다.

**수동 보완 필요 여부**: 필수. V1에서는 뉴스 신호 전체가 수동 입력 대상이다.

**최종 확정 전 검토 포인트**:
- 뉴스 신호 구조화 포맷(direction, catalyst_type, confidence 등의 enum 값)을 구현 단계에서 확정 필요
- DART 공시 자동 수집과 뉴스 신호 수동 입력은 역할이 다름 — 공시는 사실 데이터, 뉴스는 방향 해석 데이터로 분리

- 뉴스 신호 구조화 입력 형식 (입력 항목, enum 값) 최종 확정

- 기존 UI 변경 없음
