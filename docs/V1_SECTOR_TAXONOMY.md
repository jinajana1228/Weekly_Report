# V1 섹터 분류 체계

> **문서 목적**: picks[].sector 필드에 사용할 V1 섹터 분류 체계를 정의한다.
> 이 문서는 2단계 스키마 설계의 기준이 된다. Claude는 이 문서에 정의된 섹터 외의 임의 섹터 값을 사용하면 안 된다.

---

## 설계 원칙

| 원칙 | 내용 |
|------|------|
| 중복 검사 가능 | 5개 picks 섹터 중복 없음 규칙을 운영하기에 충분히 구분되어야 한다 |
| 추천 다양성 확보 | 너무 적은 섹터 수는 매주 5개 picks 구성 시 선택 폭이 좁아진다 |
| 운영 가능 범위 | V1 운영자가 종목을 섹터에 배정할 때 판단이 명확해야 한다 |
| 국내 시장 반영 | 코스피/코스닥 국내 시장 특성과 주요 테마를 반영한다 |
| ETF 포함 가능 | 개별주와 ETF 모두 하나의 섹터 체계에서 배정 가능해야 한다 |
| 최소 구조 우선 | V1은 GICS 11개 섹터/24개 산업군을 그대로 복제하지 않는다 |

---

## V1 권장 섹터 목록 (총 13개)

### 개별주 및 섹터형 ETF 공통 섹터 (10개)

| 섹터 코드 | 섹터명 | 포함 범위 (예시) |
|----------|--------|-----------------|
| `TECH` | IT·반도체 | 반도체, 디스플레이, 스마트폰, 소프트웨어, IT서비스, 인터넷 플랫폼 |
| `BATTERY` | 2차전지·전기차 | 배터리 셀, 배터리 소재, EV 부품, 충전 인프라 |
| `HEALTHCARE` | 헬스케어·바이오 | 제약, 바이오텍, 의료기기, 헬스케어 서비스 |
| `FINANCE` | 금융 | 은행, 증권, 보험, 카드, 핀테크 |
| `CONSUMER` | 소비·유통 | 음식료, 의류, 화장품, 유통, 여행, 레저 |
| `INDUSTRIAL` | 산업재·기계 | 자동차, 조선, 항공우주, 방산, 물류, 중공업 |
| `MATERIAL` | 소재·화학 | 철강, 비철금속, 석유화학, 기초소재 |
| `ENERGY` | 에너지 | 정유, LNG, 신재생에너지, 에너지 인프라 |
| `TELECOM` | 통신·미디어 | 통신사, 방송, OTT, 게임, 엔터테인먼트 |
| `REALESTATE` | 건설·부동산 | 건설, 시공, 부동산 개발, 건자재 |

### ETF 전용 섹터 (3개)

| 섹터 코드 | 섹터명 | 포함 범위 |
|----------|--------|----------|
| `ETF_DOMESTIC` | 국내지수 ETF | 코스피/코스닥 광범위 지수 추종 ETF, 국내 시가총액 상위 종합 ETF |
| `ETF_OVERSEAS` | 해외지수 ETF | 미국/글로벌 지수 추종 ETF (S&P500, NASDAQ, 신흥국 등), 해외 추종 레버리지/인버스 포함 |
| `ETF_BOND_DIV` | 채권·배당 ETF | 국내외 채권 ETF, 배당 중심 ETF, 리츠 ETF |

## ETF 섹터 배정 원칙

ETF는 복수의 자산과 섹터에 걸치기 때문에, V1에서는 아래 우선순위 원칙으로 단일 섹터를 배정한다.

### 우선순위 규칙

```
1순위: 추종 지수/전략이 특정 섹터에 명확히 집중된 경우
       → 해당 개별주 섹터로 배정
       예: 반도체 섹터 ETF → TECH
           2차전지 ETF → BATTERY
           바이오 ETF → HEALTHCARE

2순위: 추종 지수가 국내 광범위 시장인 경우
       → ETF_DOMESTIC
       예: KODEX 200, TIGER 코스닥150

3순위: 추종 지수가 해외 시장인 경우
       → ETF_OVERSEAS
       예: TIGER 미국S&P500, KODEX 나스닥100

4순위: 채권 또는 배당 중심인 경우
       → ETF_BOND_DIV
       예: KODEX 국고채10년, TIGER 고배당

우선순위 적용 후에도 판단이 불명확한 경우:
→ admin이 검수 시 배정 판단. 판단 기준은 해당 ETF의 "가장 큰 비중 노출 자산"으로 결정.
```

### ETF 섹터 배정 시 주의사항

- 동일 섹터 내 개별주와 섹터 ETF가 공존 가능하다.
  - 예: `TECH` 섹터에 삼성전자(개별주)와 KODEX 반도체(ETF)가 동시에 배정될 수 있다.
  - 이 경우 같은 주 picks에서 둘 다 선택하면 섹터 중복으로 처리된다. (의도된 설계)
- ETF 전용 섹터(`ETF_DOMESTIC`, `ETF_OVERSEAS`, `ETF_BOND_DIV`)는 개별주에 배정하지 않는다.

---

## 섹터 코드 사용 규칙

- `picks[].sector` 필드의 값은 반드시 위 13개 코드 중 하나여야 한다.
- 코드 외의 임의 값 사용 금지.
- 섹터 중복 검사는 `sector` 코드 기준으로 수행한다 (섹터명 문자열 비교 아님).
- 신규 섹터 추가 시 이 문서를 먼저 업데이트하고 admin 확인 후 적용한다.

---

## V1에 이 분류 체계가 적합한 이유

1. **13개는 중복 검사에 충분한 구분**: 5개 picks를 섹터 중복 없이 구성하기 위해 최소 5개 이상의 섹터가 필요하며, 13개는 이를 충분히 충족하면서 과도하지 않다.
2. **국내 시장 특성 반영**: 2차전지/전기차(`BATTERY`)를 별도 섹터로 분리한 것은 국내 코스피/코스닥에서 이 산업군이 차지하는 비중과 독립적 흐름을 반영한다. GICS 체계에서는 경기소비재(자동차)와 소재에 분산되어 있어 국내 운영에 불편하다.
3. **ETF 배정 규칙의 명확성**: ETF 전용 섹터 3개와 섹터 ETF의 개별주 섹터 배정 원칙을 명시하여 배정 판단이 단순하다.
4. **운영자 판단 부담 최소화**: 섹터 이름과 포함 범위가 직관적이어서 운영자가 종목 배정 시 빠르게 판단할 수 있다.

## 추후 확장 가능한 항목 (V1에서 확정하지 않음)

| 항목 | 내용 | 확장 시점 |
|------|------|----------|
| 세부 산업군 태그 | 섹터 내 세부 분류 (예: TECH > 반도체 팹리스 / 장비 / 소재) | V2 이후 |
| 복수 섹터 배정 | ETF에 여러 섹터를 동시에 배정하는 다중 태그 구조 | V2 이후 (DB 전환 시 용이) |
| 섹터 간 유사도 매핑 | GICS 또는 KRX 섹터와의 매핑 테이블 | 데이터 소스 확정 후 |
| 지역 태그 | 해외 추종 ETF의 추종 지역 세분화 (미국/유럽/신흥국) | V2 이후 |
| 자동 섹터 배정 | 종목 코드 기반 자동 섹터 분류 로직 | 데이터 소스 확정 후 |

---

> 이 섹터 체계는 V1 운영 원칙이다. 2단계 스키마 설계 시 picks[].sector 허용값은 이 문서의 13개 코드를 기준으로 한다.
