# V1 종목 추천 로직 설계 문서

> **문서 목적**: 매주 발행하는 1개월형 주간 추천 리포트의 종목 추천 로직을 어떤 구조와 절차로 운영할지 정의한다.
> 이 문서는 구현 전 설계 검토 단계의 산출물이다. 실제 점수 가중치 확정, 데이터 소스 최종 확정, 코드 구현은 이 문서에서 하지 않는다.

---

## 1. 1개월형 종목 추천의 기본 철학

### 핵심 명제

> "2~4주 안에 가시적인 가격 반응을 기대할 수 있는 근거가 있는 종목을 선택한다."

이 서비스의 추천은 다음 3가지 조건이 동시에 충족될 때 가장 이상적이다.

1. **단기 촉매가 있다**: 실적 발표, 제품 출시, 정책 수혜, 수급 전환 등 2~4주 내 가시적 이벤트가 존재한다.
2. **가격 위치가 합리적이다**: 현재 가격이 관심 구간(watch zone) 내에 있거나 최근 과열이 아닌 상태다.
3. **시장 분위기와 맞는다**: 이번 주 시장의 섹터 선호 방향과 추천 종목의 섹터가 충돌하지 않는다.

### 3~6개월형 추천과의 차이

| 비교 항목 | 1개월형 (이 서비스) | 3~6개월형 |
|----------|------------------|----------|
| 핵심 평가 축 | 단기 촉매 + 가격 위치 + 수급 | 장기 성장성 + 밸류에이션 |
| 제외 기준 | 촉매 없음, 가격 과열, 수급 이탈 | 재무 불건전, 산업 구조적 쇠퇴 |
| 리스크 관점 | 이번 2~4주 내 발생 가능한 이벤트 리스크 | 장기 구조적 리스크 |
| 업데이트 주기 | 매주 (시장 분위기 변화 반영) | 분기 또는 반기 |
| 재무 분석 비중 | 기본 안전성 필터 수준 (극단적 위험 제거) | 심층 밸류에이션 분석 |

## 2. 추천 로직의 핵심 평가축

아래 5개 평가축이 1개월형 추천의 기본 판단 구조다.

### 평가축 1: 단기 촉매 (Short-term Catalyst)

**정의**: 2~4주 내에 가시적으로 주가에 영향을 줄 수 있는 구체적 이벤트 또는 변화.

**왜 1개월형에서 가장 중요한가**: 1개월 안에 가격이 반응하려면 반응의 트리거가 있어야 한다. 촉매 없이 "좋은 기업이니 오를 것"이라는 논리는 1개월 타임라인에서 작동하지 않는다.

**어떤 종목에 유리하게 작용하는가**: 실적 발표 직전 기업, 정책 수혜 가시화 기업, 제품/서비스 출시 임박 기업, 수급 전환 초기 기업.

**어떤 데이터로 판단 가능한가**: 기업 실적 발표 일정(공시 캘린더), 주요 정책/규제 일정, 산업별 이벤트 캘린더, 뉴스 신호.

> ⚠️ **정량화 기준 미확정**: "촉매가 있다"의 최소 요건을 점수화하는 기준은 구현 단계에서 결정한다.

---

### 평가축 2: 가격 위치 (Price Position)

**정의**: 현재 주가가 관심 구간(watch zone) 대비 어디에 있는가. 최근 급등 과열 상태는 아닌가.

**왜 1개월형에서 중요한가**: 좋은 촉매가 있어도 이미 가격이 선반영된 상태라면 추가 상승 여력이 적다. 1개월 타임라인에서는 진입 가격이 수익률에 직접적으로 영향을 미친다.

**어떤 종목에 유리하게 작용하는가**: 최근 조정 후 관심 구간 하단에 있는 종목, 박스권 하단에서 반등 시도 중인 종목, 아직 시장에 덜 알려진 촉매를 가진 종목.

**어떤 데이터로 판단 가능한가**: 주가 시계열, 52주 고/저가 대비 위치, 이동평균선 대비 위치.

> ⚠️ **정량화 기준 미확정**: 가격 위치의 "과열" 판단 기준(예: 52주 고가 대비 몇 % 이내 등)은 구현 단계에서 결정한다.

---

### 평가축 3: 수급 신호 (Supply & Demand Signal)

**정의**: 기관/외국인 순매수 동향, 거래량 변화 등 실제 자금 흐름이 해당 종목에 유리하게 움직이고 있는가.

**왜 1개월형에서 중요한가**: 1개월 타임라인은 수급의 지속성이 결과에 직결된다. 외국인/기관이 이탈 중인 종목은 단기에 회복이 어렵다. 반대로 수급 전환 초기 신호는 강력한 선행 지표다.

**어떤 종목에 유리하게 작용하는가**: 외국인 순매수 전환 초기 종목, 기관 연속 매수 종목, 거래량이 평균 대비 증가 추세인 종목.

**어떤 데이터로 판단 가능한가**: 외국인/기관/개인 순매수 데이터 (KRX 등), 거래량 데이터.

> ⚠️ **정량화 기준 미확정**: 수급 신호의 "전환" 기준(예: n일 연속 순매수 등)은 구현 단계에서 결정한다.

---

### 평가축 4: 시장 적합도 (Market Fit)

**정의**: 이번 주 시장 분석에서 도출된 `favored_sectors`와 추천 종목의 섹터가 맞는가. 현재 시장 분위기(Risk-on/off, 대형주/소형주 선호 등)가 해당 종목에 유리한가.

**왜 1개월형에서 중요한가**: 좋은 개별 종목이라도 시장 전체가 해당 섹터를 기피하는 방향이라면 1개월 내 성과가 억눌린다. 이 서비스의 추천은 "지금 이 시장에서 통하는 종목"을 뽑는 것이다.

**어떤 종목에 유리하게 작용하는가**: `favored_sectors`에 해당하는 섹터의 종목, 현재 Risk-on 환경에서는 성장주/모멘텀, Risk-off 환경에서는 배당/방어주.

**어떤 데이터로 판단 가능한가**: 시장 분석 문서의 favored/cautious 섹터 판단 결과, VIX 수준, 외국인 전체 순매수 방향.

> ⚠️ **정량화 기준 미확정**: 시장 적합도의 점수화 방식은 구현 단계에서 결정한다.

---

### 평가축 5: 재무 안전성 (Financial Safety)

**정의**: 이 종목이 추천 기간(2~4주) 내 재무적 이슈로 급격히 훼손될 위험이 없는가. 기본적인 재무 건전성을 확인하는 최소 필터다.

**왜 1개월형에서 중요한가**: 1개월형은 장기 밸류에이션 분석 서비스가 아니다. 따라서 심층 재무 분석보다는 "재무 안전성을 해치는 극단적 위험 요인만 걸러내는" 필터 역할을 한다. 점수 경쟁이 아니라 최소 통과 기준이다.

**어떤 데이터로 판단 가능한가**: 영업이익(흑자 여부), 부채비율(극단적 고레버리지 여부), 현금흐름(영업활동현금 흑자 여부).

> ⚠️ **정량화 기준 미확정**: 재무 안전성 필터의 구체적 수치 기준은 구현 단계에서 결정한다.

---

## 3. 메인 추천 5개 선정 절차

### Step 1: 추천 유니버스 정의

- 코스피 및 코스닥 상장 기업 및 상장 ETF를 대상으로 한다.
- V1에서는 관리자가 수동으로 후보 목록을 작성하거나, 일정 기준을 충족하는 종목군에서 시작한다.
- 유니버스 규모는 확정되지 않았다.

> ⚠️ **미확정**: 유니버스 구성 방식(전체 상장 종목 자동 스캔 vs 사전 선정 종목군 vs 섹터별 대표주 + ETF 고정 목록)은 구현 단계에서 결정한다.

---

### Step 2: 제외 필터 적용 (Hard Filter)

아래 필터에 해당하는 종목은 메인 추천 후보에서 강제 제외한다.
필터는 "반드시 넘어야 하는 문턱"이며, 점수로 보완할 수 없다.

| 제외 필터 | 기준 설명 | 수치 기준 |
|----------|----------|---------|
| 영업 적자 기업 | 최근 연간 또는 최근 2개 분기 기준 영업적자 | ⚠️ 기준 기간 미확정 |
| 신규 상장 | 상장 후 일정 기간 미경과 종목 | ⚠️ 기간 기준 미확정 |
| 동전주 | 주가가 일정 기준 이하인 저가 종목 | ⚠️ 수치 기준 미확정 |
| 저유동성 | 일평균 거래량 또는 거래대금이 일정 기준 미달 | ⚠️ 수치 기준 미확정 |
| 관리종목 | 거래소 지정 관리종목 | 한국거래소 지정 여부 확인 |
| 거래정지 종목 | 현재 거래정지 상태 | 상태 확인 |
| 감사의견 비적정 | 감사보고서 의견이 적정 미만 | 감사보고서 확인 |
| 과열 이슈 종목 | 거래소/금감원 투자주의/경고/위험 지정 종목 | 공식 지정 여부 확인 |
| 지나친 테마성 종목 | 구체적 실적 근거 없이 단기 이슈만으로 급등한 종목 | ⚠️ 판단 기준 미확정 (사람 판단 필요) |
| 최근 급등 과열 종목 | 단기간 내 급격히 상승하여 추가 상승 여력이 낮은 구간 | ⚠️ 수치 기준 미확정 |

> ⚠️ **수치 기준 미확정 항목**: 위 표에서 미확정으로 표시된 항목의 구체적 수치는 구현 단계에서 결정한다.

---

### Step 3: 개별 후보 평가

Step 2를 통과한 후보 종목에 대해 5개 평가축(단기 촉매 / 가격 위치 / 수급 신호 / 시장 적합도 / 재무 안전성)으로 평가한다.

- 평가 방식(점수화 vs 정성 판단 vs 혼합)은 구현 단계에서 결정한다.
- 각 축의 가중치는 구현 단계에서 결정한다.
- 재무 안전성은 최소 통과 기준으로 다루며 점수 경쟁 항목이 아니다.

---

### Step 4: 섹터 분산 강제 적용

확정 원칙: 메인 추천 5개는 동일 섹터 중복이 없어야 한다. 13개 섹터 코드(V1_SECTOR_TAXONOMY.md 참조) 기준으로 판단한다.

적용 방식:

```
1. 평가 점수 상위 후보들을 순서대로 선택한다.
2. 이미 선택된 종목의 섹터와 중복되는 종목은 건너뛴다.
3. 5개가 채워질 때까지 반복한다.
4. 점수가 유사한 경우 섹터 분산 원칙에 의해 하위 점수 종목이 선택될 수 있다.
```

> ⚠️ **미확정**: 점수 차이가 클 경우 섹터 중복을 허용할 예외 케이스가 있는지는 운영 원칙 확정 필요. V1 원칙상 예외 없음으로 시작한다.

---

### Step 5: ETF 포함 여부 판단

- ETF는 개별주와 동일한 유니버스에서 평가한다.
- ETF 전용 섹터(`ETF_DOMESTIC`, `ETF_OVERSEAS`, `ETF_BOND_DIV`) 또는 섹터형 ETF(`TECH` 등)로 섹터 배정된다.
- ETF가 포함될 경우 해당 주에 같은 섹터의 개별주는 메인 추천에 포함되지 않는다.
- ETF를 메인 추천에 포함시킬 때는 "이번 주 이 ETF를 추천하는 이유"가 구체적이어야 한다. 단순히 분산 효과나 안전성만을 이유로 추천하는 것은 부족하다.

---

### Step 6: 중복 추천 정책 검토

`admin/overlap_history.json`의 최근 3개 에디션 이력을 확인한다.

| 상황 | 처리 방식 |
|------|----------|
| 직전 주(n-1) 메인 추천 종목 | 원칙적 제외. 후보에서 제거. |
| 2주 전(n-2) 메인 추천 종목 | 허용. 단, 재추천 사유가 명확해야 함. |
| 3주 연속 추천 시도 (n-1, n-2 모두 추천) | 강제 제외. 예외 없음. |
| 직전 주 제외 예외 적용 | admin 검수 시 예외 승인 필요. 승인 사유 기록 필수. |

---

### Step 7: 최종 5개 확정 + same_sector_alternatives 산출

- 메인 5개 확정 후, 각 메인 종목의 섹터 내에서 same_sector_alternatives 2개를 추가 선정한다.
- same_sector_alternatives는 메인 추천 절차(Step 1~6)를 거쳤으나 메인에 선정되지 않은 종목 중에서 선택한다.
- same_sector_alternatives에는 중복 추천 정책을 적용하지 않는다. (이 원칙은 정책 확정 필요 항목임)

---

## 4. same_sector_alternatives 2개 선정 절차

### 목적

메인 추천 종목을 직접 매수하기 어려운 독자 또는 더 넓은 관점을 원하는 독자를 위해 동일 섹터 내 추가 후보를 제공한다.

### 선정 기준

1. 메인 추천 종목과 **동일 섹터(sector 코드 기준)**여야 한다.
2. 제외 필터(Step 2)를 통과한 종목이어야 한다.
3. 메인 추천 종목과 **동일 종목이 아니어야** 한다 (당연히).
4. 메인 추천 종목과 완전히 동일한 사업 구조를 가진 종목보다는 **보완적 포지션**을 가진 종목이 더 적합하다.
   - 예: 삼성전자(DRAM)가 메인이면, 같은 TECH 섹터 내 NAND 또는 시스템반도체 쪽 종목이 alternatives로 더 의미있다.
5. 본 추천 리포트에서 충분한 설명 없이 단순히 이름과 한 줄 이유만 제공된다. 따라서 alternatives는 추가 설명 없이도 이해할 수 있는 이유가 있어야 한다.

> ⚠️ **미확정**: same_sector_alternatives에도 중복 추천 정책(직전 주 제외 등)을 동일하게 적용할지는 운영 원칙 추가 확정 필요.

---

## 5. 개별주와 ETF를 함께 추천할 때의 원칙

### 같은 주에 개별주와 ETF가 함께 메인에 포함되는 경우

| 원칙 | 내용 |
|------|------|
| 섹터 분리 | 개별주와 ETF는 반드시 서로 다른 섹터 코드를 가져야 한다. 동일 섹터에서 개별주와 그 섹터의 ETF를 동시에 메인에 넣는 것은 실질적 중복이다. |
| ETF 추천 사유의 차별화 | ETF를 추천하는 이유가 "개별 종목 리스크를 줄이기 위해"라는 단순 설명은 부족하다. 이번 2~4주에 특정 ETF를 선택하는 구체적 시장 이유가 있어야 한다. |
| ETF 추가 설명 항목 | 개별주보다 ETF는 추가로 설명해야 하는 항목이 있다 (아래 참조). |

### ETF 추천 시 개별주보다 추가로 설명해야 하는 핵심 항목

| 항목 | 이유 |
|------|------|
| 추종 지수/전략의 특성 | ETF는 무엇을 추종하는지 모르면 가격 움직임을 이해할 수 없다. |
| 환헤지 여부 | 해외 ETF는 환율 변동이 수익률에 직접 영향을 준다. |
| 레버리지/인버스 여부 | 레버리지 ETF는 일반 종목과 전혀 다른 위험 구조를 가진다. |
| 주요 구성 종목/비중 | ETF의 실제 노출이 어디에 있는지 독자가 이해할 수 있어야 한다. |
| 운용사 및 수수료 | 유사한 ETF 간 선택의 기준이 된다. |
| ETF 전용 리스크 | 일반 종목 리스크와 다른 ETF 고유의 리스크(괴리율, 유동성 위험 등)를 별도 설명한다. |

## 6. 제외 필터 체계 정리

### Hard Filter (점수로 통과 불가)

아래 필터를 통과하지 못하면 메인 추천, same_sector_alternatives 모두에서 제외된다.

| 필터 | 확정 여부 |
|------|----------|
| 관리종목 지정 | ✅ 확정 |
| 거래정지 상태 | ✅ 확정 |
| 감사의견 비적정 | ✅ 확정 |
| 거래소 과열/위험 지정 | ✅ 확정 |
| 영업 적자 | ⚠️ 기준 기간 미확정 |
| 신규 상장 | ⚠️ 기간 기준 미확정 |
| 동전주 | ⚠️ 수치 기준 미확정 |
| 저유동성 | ⚠️ 수치 기준 미확정 |

### Soft Filter (점수에 반영하되 자동 제외는 아님)

아래 항목은 점수에 부정적 영향을 주지만, 사람 판단으로 예외 허용 가능하다.

| 필터 | 처리 방식 |
|------|----------|
| 최근 급등 과열 구간 | 가격 위치 평가축에서 낮은 점수. admin 검수 시 추가 확인. |
| 지나친 테마성 종목 | 촉매 평가축에서 "촉매의 구체성" 항목으로 판단. 실적 근거 없는 테마는 낮은 점수. |
| 직전 주 메인 추천 | 중복 추천 정책에 따라 원칙적 제외. 예외 승인 시 admin 사유 기록 필요. |

## 7. 중복 추천 정책이 추천 로직에 반영되는 방식

`admin/overlap_history.json`에서 최근 3개 에디션의 메인 추천 티커 목록을 읽어 아래와 같이 적용한다.

```
추천 로직에서의 적용 순서:

1. 유니버스에서 hard filter 적용 후 후보군 확보
2. 후보군 중 "직전 주(n-1) 메인 추천" 종목 → 후보에서 제거 (원칙적 제외)
3. 후보군 중 "2주 전(n-2)과 직전 주(n-1) 모두 메인 추천이었던 종목" → 강제 제외 (3주 연속 금지)
4. 나머지 후보군에서 평가 진행
5. 최종 5개 선정 후, overlap_history와 대조하여 위반 여부 재확인
6. 위반 있으면 다음 후보로 교체
```

직전 주 제외 예외 적용 시:
- admin 검수 화면에서 예외 승인 표시 필요
- 예외 사유 텍스트 기록 필수
- 예외 적용 종목은 draft 기준일 기준으로 `approval.json`에 흔적이 남아야 함 (구체적 필드 구조는 추후 확정)

## 8. 점수화 가능 항목 vs 사람 판단이 필요한 항목

### 점수화 가능 항목 (정량 데이터 기반)

| 항목 | 점수화 방법 예시 |
|------|--------------|
| 가격 위치 | 52주 고가 대비 현재 위치 비율, 이동평균선 이격도 |
| 수급 신호 | 외국인/기관 순매수 연속 일수, 순매수 금액 변화율 |
| 업종 상대강도 | 섹터의 코스피 대비 주간 초과수익률 |
| 재무 안전성 | 영업이익 흑자 여부, 부채비율 구간 |
| 중복 추천 이력 | overlap_history.json 기반 자동 플래그 |

### 사람 판단이 필요한 항목

| 항목 | 이유 |
|------|------|
| 촉매의 질과 구체성 | "실적 발표"가 있어도 어닝 서프라이즈 가능성이 있는지는 수치만으로 판단 불가 |
| 테마성 판단 | 단기 이슈만으로 급등한 종목인지는 맥락 이해가 필요 |
| ETF 추종 지수의 이번 주 적합성 | 시장 분위기와 ETF의 노출 구조가 맞는지는 종합 판단 필요 |
| same_sector_alternatives의 보완성 | 메인 종목과의 관계에서 "다양성"이 있는지는 사람 판단 필요 |
| 중복 추천 예외 승인 | 예외 인정 사유의 타당성은 사람이 판단해야 함 |

## 9. 확정 현황

### 확정된 원칙

| 항목 | 확정 내용 |
|------|----------|
| 추천 관점 기간 | 2~4주 (1개월형) |
| 메인 추천 수 | 5개 |
| 섹터 중복 금지 | 메인 5개는 섹터 코드 기준 중복 없음 |
| 섹터 분류 체계 | 13개 섹터 코드 (V1_SECTOR_TAXONOMY.md) |
| same_sector_alternatives | 메인 1개당 2개, 동일 섹터 내 |
| 중복 추천 직전 주 제외 | 원칙적 제외, 예외 시 admin 승인 필요 |
| 3주 연속 추천 금지 | 강제 제외, 예외 없음 |
| Hard Filter 확정 항목 | 관리종목, 거래정지, 감사의견 비적정, 거래소 과열/위험 지정 |
| ETF 추가 설명 항목 | 추종 지수, 환헤지, 레버리지, 구성 종목, 수수료, ETF 전용 리스크 |

### 정책 확정이 더 필요한 항목

| 항목 | 미확정 이유 | 확정 필요 시점 |
|------|-----------|--------------|
| 유니버스 구성 방식 | 자동 스캔 vs 수동 선정 방식 미결 | 데이터 소스 설계 단계 |
| 평가축 가중치 | 실제 수치 가중치 미확정 | 구현 단계 |
| Hard Filter 수치 기준 (적자/신규상장/동전주/저유동성) | 운영 기준값 미확정 | 구현 단계 |
| same_sector_alternatives 중복 정책 적용 여부 | 운영 원칙 추가 확정 필요 | 운영 원칙 결정 시 |
| 예외 연속 추천 허용 조건 | 구체적 조건 기준 미확정 | 운영 경험 축적 후 보정 |
| 점수 계산 방식 (자동/반자동/수동) | V1 운영 방식 확정 필요 | 구현 단계 |

> 이 문서는 V1 종목 추천 로직의 설계 기준이다.
> 실제 데이터 수집 방식, 점수 계산 엔진 구현, 가중치 수치 확정은 이 문서에서 하지 않는다.
